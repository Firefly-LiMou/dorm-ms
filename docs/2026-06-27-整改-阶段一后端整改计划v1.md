### 阶段A：基础层（无依赖，最先完成）

#### A-01：BusinessException 添加业务错误码常量

**目标**：为不同类型业务错误定义标准错误码，Service层抛出异常时使用

**修改文件**：`com.huuc.dormitory.common.exception.BusinessException.java`

**修改内容**：
- 添加业务错误码常量定义（400/401/403/404/409）
- 保留现有构造函数不变

**错误码定义**：
```
400 = 参数错误
401 = 认证失败（用户名密码错误、账号禁用）
403 = 权限不足
404 = 资源不存在（楼栋/房间/床位/用户/记录不存在）
409 = 状态冲突（用户名已存在、编号已存在、床位已占用、已有在住记录、已有待审批申请）
500 = 系统内部错误（默认值）
```

**验收标准**：
- 常量命名为 `CODE_BAD_REQUEST`、`CODE_UNAUTHORIZED`、`CODE_FORBIDDEN`、`CODE_NOT_FOUND`、`CODE_CONFLICT`、`CODE_ERROR`
- 不改变现有构造函数签名
- 不改变现有代码行为（现有代码仍使用默认500）

#### B-02：GlobalExceptionHandler 补充常见异常处理

**目标**：捕获常见Spring MVC异常，返回明确错误信息而非"系统繁忙"

**修改文件**：`com.huuc.dormitory.common.exception.GlobalExceptionHandler.java`

| 异常类型 | 返回code | 返回msg |
|----------|----------|---------|
| HttpMessageNotReadableException | 400 | "请求格式错误" |

#### B-03：OperLogAspect 修复失败操作不记录问题

**目标**：业务方法抛出异常时，仍记录操作日志

**修改文件**：`com.huuc.dormitory.common.aop.OperLogAspect.java`

**修改内容**：
- 将 `joinPoint.proceed()` 包裹在 try-catch 中
- 无论成功或失败，都记录操作日志
- 失败时在日志中记录异常信息（截取前500字符）

**修改逻辑**：
```
try {
    result = joinPoint.proceed();
    // 记录成功日志
} catch (Throwable e) {
    // 记录失败日志（包含异常信息）
    throw e;  // 继续抛出，不影响异常处理流程
} finally {
    // 统一写入日志表
}
```

**验收标准**：
- 成功操作记录到 sys_oper_log 表
- 失败操作也记录到 sys_oper_log 表
- 异常继续向上传播，不被吞掉
- 日志记录失败仍不影响主业务

### 阶段C：拦截器层（依赖阶段B）

#### C-01：PermissionInterceptor 返回JSON格式

**目标**：权限校验失败时返回统一JSON格式，而非HTML

**修改文件**：`com.huuc.dormitory.interceptor.PermissionInterceptor.java`

**修改内容**：
- 注入 ObjectMapper（通过@Component + @Autowired）
- 权限校验失败时，设置 `response.setContentType("application/json;charset=UTF-8")`、`response.setStatus(403)`，写入JSON `{"code":403,"msg":"无权限访问","data":null}`
- 移除 `response.sendError()` 调用

**前置条件**：
- 在spring-mvc.xml中将PermissionInterceptor改为Spring Bean注册（而非内联bean）
- 或者在PermissionInterceptor类上添加@Component注解（spring-mvc.xml已扫描interceptor包）

**验收标准**：
- 前端AJAX请求收到JSON而非HTML
- HTTP状态码为403
- 响应体为 `{"code":403,"msg":"无权限访问","data":null}`

#### C-02：统一未登录响应格式

**目标**：LoginInterceptor 和 AuthController 对未登录场景返回一致的响应格式

**修改文件**：
- `com.huuc.dormitory.interceptor.LoginInterceptor.java`
- `com.huuc.dormitory.controller.AuthController.java`

**方案**：
- LoginInterceptor 判断请求是否为AJAX请求（检查 `X-Requested-With: XMLHttpRequest` 头）
  - 如果是AJAX请求：返回JSON `{"code":401,"msg":"未登录","data":null}`，HTTP 401
  - 如果是普通请求：重定向至登录页（保持现有行为）
- AuthController 的 `/currentUser` 和 `/needChangePassword` 保持现有JSON 401返回不变

**验收标准**：
- 前端AJAX请求未登录时收到JSON 401
- 浏览器直接访问未登录时重定向到登录页
