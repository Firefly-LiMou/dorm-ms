# 阶段四：宿舍基础信息管理模块开发计划

## 一、模块概述

### 1.1 功能范围
- 楼栋管理：管理员CRUD、宿管查看
- 房间管理：管理员CRUD、宿管查看
- 床位管理：管理员新增/批量初始化/状态管理、宿管查看

### 1.2 数据层级关系
```
楼栋（dorm_building）
  └── 房间（dorm_room）
        └── 床位（dorm_bed）
```

### 1.3 角色权限矩阵
| 功能 | 管理员 | 宿管 | 学生 |
|------|--------|------|------|
| 楼栋列表 | 全部 | 本人负责楼栋 | - |
| 楼栋新增/编辑/删除 | ✓ | - | - |
| 房间列表 | 全部 | 本楼栋 | - |
| 房间新增/编辑/删除 | ✓ | - | - |
| 床位列表 | 全部 | 本楼栋房间 | - |
| 床位新增/批量初始化 | ✓ | - | - |
| 床位状态修改 | ✓ | - | - |

## 三、DTO设计

### 3.1 BuildingDTO（楼栋新增/编辑）

```java
@Getter
@Setter
@NoArgsConstructor
@ToString
public class BuildingDTO {
    private Long buildingId;        // 编辑时必填
    @NotBlank(message = "楼栋编号不能为空")
    private String buildingNo;
    @NotBlank(message = "楼栋名称不能为空")
    private String buildingName;
    @NotNull(message = "楼层总数不能为空")
    @Min(value = 1, message = "楼层总数至少为1")
    private Integer floorCount;
    private String area;
    private Long managerId;
    private String remark;
}
```

### 3.2 RoomDTO（房间新增/编辑）

```java
@Getter
@Setter

@ToString

    private Long buildingId;
    @NotNull(message = "所在楼层不能为空")

    private Integer floorNum;
    @NotNull(message = "额定床位数不能为空")
```

## 六、Service层设计

### 6.1 BuildingService

```java
public interface BuildingService {
    BuildingVO getBuildingById(Long buildingId);
    PageInfo<BuildingVO> getBuildingList(DormBuilding query, Integer pageNum, Integer pageSize);
    List<BuildingVO> getBuildingsByManagerId(Long managerId);
    void addBuilding(BuildingDTO dto, Long operatorId);
    void updateBuilding(BuildingDTO dto, Long operatorId);
    void deleteBuilding(Long buildingId, Long operatorId);
}
```

**业务规则**：
- deleteBuilding：校验楼栋下是否存在房间，存在则抛出异常"该楼栋下存在房间，无法删除"，不存在则逻辑删除

### 6.2 RoomService

```java
public interface RoomService {
    RoomVO getRoomById(Long roomId);
    PageInfo<RoomVO> getRoomList(DormRoom query, Integer pageNum, Integer pageSize);
    List<RoomVO> getRoomsByBuildingId(Long buildingId);
    void addRoom(RoomDTO dto, Long operatorId);
    void updateRoom(RoomDTO dto, Long operatorId);
    void deleteRoom(Long roomId, Long operatorId);
}
```

**业务规则**：
- addRoom：校验楼栋是否存在，校验楼层是否在楼栋范围内，校验同一楼栋下房间号是否重复
- deleteRoom：校验房间下是否存在床位，存在则抛出异常"该房间下存在床位，无法删除"，不存在则逻辑删除

### 6.3 BedService

```java
public interface BedService {
    BedVO getBedById(Long bedId);
    List<BedVO> getBedsByRoomId(Long roomId);
    List<BedVO> getFreeBedsByRoomId(Long roomId);
    void addBed(BedDTO dto, Long operatorId);
    void batchAddBeds(BatchBedDTO dto, Long operatorId);
    void updateBedStatus(Long bedId, Integer status);
}
```

**业务规则**：
- batchAddBeds：
  1. 校验房间是否存在
  2. 校验床位数是否合法（大于0）
  3. 生成床位号：1号床、2号床...N号床
  4. 批量插入床位记录
  5. 失败时整体回滚（@Transactional）
- updateBedStatus：校验床位是否存在

## 七、Controller层设计

### 7.1 管理员端

**AdminBuildingController**（/admin/building）
- GET /list - 楼栋列表页面
- GET /page - 查询楼栋列表（分页）
- GET /{buildingId} - 获取楼栋详情
- POST /add - 新增楼栋
- POST /update - 编辑楼栋
- POST /delete/{buildingId} - 删除楼栋

**AdminRoomController**（/admin/room）
- GET /list - 房间列表页面
- GET /page - 查询房间列表（分页）
- GET /building/{buildingId} - 获取楼栋下房间列表
- GET /{roomId} - 获取房间详情
- POST /add - 新增房间
- POST /update - 编辑房间
- POST /delete/{roomId} - 删除房间

**AdminBedController**（/admin/bed）
- GET /room/{roomId} - 获取房间下床位列表
- GET /free/{roomId} - 获取房间下空闲床位列表
- GET /{bedId} - 获取床位详情
- POST /add - 新增床位
- POST /batch - 批量初始化床位
- POST /status/{bedId} - 修改床位状态

### 7.2 宿管端

**DormBuildingController**（/dorm/building）
- GET /list - 本人负责楼栋列表

**DormRoomController**（/dorm/room）
- GET /building/{buildingId} - 楼栋下房间列表

**DormBedController**（/dorm/bed）
- GET /room/{roomId} - 房间下床位列表

## 十一、异常处理

| 场景 | 异常信息 |
|------|----------|
| 楼栋不存在 | "楼栋不存在" |
| 楼栋下存在房间 | "该楼栋下存在房间，无法删除" |
| 房间不存在 | "房间不存在" |
| 房间下存在床位 | "该房间下存在床位，无法删除" |
| 同楼栋房间号重复 | "该楼栋下已存在相同房间号" |
| 楼层超出范围 | "楼层超出楼栋范围" |
| 床位不存在 | "床位不存在" |
| 房间不存在（批量） | "房间不存在" |
| 床位数不合法 | "床位数量必须大于0" |
