# 高校公寓管理系统 — 前端页面风格约束 v1

> 文档版本：v1 | 最后更新：2026-06-27
> 适用范围：所有前端页面（JSP + Bootstrap 5.3.3 + jQuery 3.6.0 + FontAwesome 6.5.1）
> 前置依赖：无

---

## 一、设计哲学

本系统采用**杂志风亮色主题**。核心原则：大量留白、克制的强调色、衬线标题与无衬线正文的字体对比、等宽字体承载数据与标签、半调网点底纹增加纸张质感。视觉目标为"可阅读的管理系统"，而非"装饰性仪表盘"。

---

## 二、基础设计令牌（Design Tokens）

### 2.1 色彩系统

所有颜色必须使用 `rgb(x, x, x)` 或 `rgba(x, x, x, x)` 格式，禁止使用 hex（`#xxx`）或关键字（`red`、`blue`）。

| 令牌名 | 用途 | 色值 | CSS 变量 |
|:---|:---|:---|:---|
| `--bg` | 页面底层背景（半调网点底色） | `rgb(250, 249, 245)` | `--bg` |
| `--surface` | 卡片、面板、弹窗等浮层背景 | `rgba(255, 255, 255, 0.92)` | `--surface` |
| `--fg` | 主文字、标题 | `rgb(45, 36, 28)` | `--fg` |
| `--muted` | 次级文字、标签、眉标 | `rgb(122, 112, 103)` | `--muted` |
| `--border` | 边框、分割线 | `rgb(235, 230, 223)` | `--border` |
| `--accent` | 主强调色（锗红） | `rgb(196, 69, 58)` | `--accent` |
| `--accent-2` | 辅助强调色（暖橙） | `rgb(212, 132, 90)` | `--accent-2` |
| `--hover-bg` | 行/项 hover 底色 | `rgb(245, 244, 237)` | `--hover-bg` |
| `--stripe` | 表格奇数行条纹色 | `rgba(0, 0, 0, 0.025)` | `--stripe` |
| `--col-line` | 表格列间竖线色 | `rgb(240, 238, 230)` | `--col-line` |
| `--accent-soft` | accent 的 10% 透明度版本 | `color-mix(in oklch, var(--accent) 10%, transparent)` | `--accent-soft` |
| `--accent-2-soft` | accent-2 的 12% 透明度版本 | `color-mix(in oklch, var(--accent-2) 12%, transparent)` | `--accent-2-soft` |
| `--fg-soft` | fg 的 6% 透明度版本 | `color-mix(in oklch, var(--fg) 6%, transparent)` | `--fg-soft` |

**色彩使用规则：**
- 每个屏幕最多使用 2 处 accent 色（如眉标 + 主按钮、侧栏激活态 + 标题强调）
- 不得出现紫色/紫色渐变背景
- 不得出现暖米色/奶油色/桃色/粉色/橙棕色整页背景渐变
- 半调网点底色固定为 `rgb(250, 249, 245)`，其上的网点色固定为 `rgb(237, 234, 229)`

### 2.2 字体系统

| 用途 | 字体族 | 字号 | 字重 | 行高 | 字间距 |
|:---|:---|:---|:---|:---|:---|
| 显示层（h1） | Georgia, Noto Serif SC, Source Han Serif SC, SimSun, serif | `32px` | 700 | 1.15 | `-0.01em` |
| 显示层（h2） | 同上 | `22px` | 700 | 1.2 | `-0.01em` |
| 显示层（h3） | 系统无衬线 | `16px` | 600 | 1.3 | `-0.005em` |
| 正文 | PingFang SC, Microsoft YaHei, -apple-system, system-ui, sans-serif | `14px` | 400 | 1.6 | `0` |
| 数据/标签/眉标 | SF Mono, JetBrains Mono, Consolas, ui-monospace, monospace | `11px` | 500 | 1.4 | `0.04em` |
| 元信息/分页 | 同上 | `12px` | 400 | 1.4 | `0` |
| 大号数据 | Georgia, Noto Serif SC, serif | `36px` | 700 | 1 | `-0.02em` |

**字体使用规则：**
- h1、h2 必须使用衬线字体族（`var(--font-display)`），不得使用 Inter / Roboto / Arial 等无衬线字体作为显示层
- 正文使用系统无衬线字体族（`var(--font-body)`），禁止在正文中使用衬线字体
- 所有数字、编号、价格、统计值必须使用等宽字体族（`var(--font-mono)`），并设置 `font-variant-numeric: tabular-nums`
- 眉标（eyebrow）使用等宽字体、11px、`letter-spacing: 0.06em`、`text-transform: uppercase`

### 2.3 间距系统

基于 8px 网格，所有间距必须使用以下枚举值：

| 令牌名 | 值 | 用途 |
|:---|:---|:---|
| `--gap-xs` | `6px` | 极小间距（标签与输入框之间） |
| `--gap-sm` | `10px` | 小间距（表格行内 padding、按钮内 gap） |
| `--gap-md` | `16px` | 中间距（卡片内间距、表格单元格 padding） |
| `--gap-lg` | `24px` | 大间距（区块间、侧栏 padding） |
| `--gap-xl` | `40px` | 超大间距（页面级 section 间距） |

**间距规则：**
- 页面顶部到第一个内容块：`40px`
- section 与 section 之间：`40px`
- 卡片内部 padding：`24px`
- 表格单元格 padding：`12px 16px`
- 按钮内 padding：`9px 18px`（常规）、`5px 12px`（小号）

### 2.4 圆角

| 令牌名 | 值 | 用途 |
|:---|:---|:---|
| `--radius` | `8px` | 按钮、输入框、下拉框、小型面板 |
| `--radius-lg` | `12px` | 卡片、数据面板、弹窗、大面板 |

**圆角规则：**
- 所有可交互元素必须有圆角，不得出现直角按钮或直角输入框
- 圆角值只能使用 `8px` 或 `12px` 两个枚举值，禁止出现 `4px`、`6px`、`16px`、`20px` 等中间值
- pill/badge 使用 `999px`（完全圆形端点）

## 三、半调网点底纹

### 3.1 实现方式

```css
body {
  background:
    radial-gradient(circle, rgb(237, 234, 229) 0.8px, transparent 0.8px),
    rgb(250, 249, 245);
  background-size: 16px 16px;
}
```

### 3.2 参数规范

| 参数 | 值 | 说明 |
|:---|:---|:---|
| 网点形状 | 圆形（`circle`） | 不得使用椭圆或方形 |
| 网点半径 | `0.8px` | 直径 1.6px，肉眼可见但不抢视觉 |
| 网点颜色 | `rgb(237, 234, 229)` | 比底色深约 5%，不可过深 |
| 网点间距 | `16px` | 网格密度固定，不可调整 |
| 底色 | `rgb(250, 249, 245)` | 与 `--bg` 一致 |

### 3.3 透出规则

所有浮层面板（卡片、侧栏、头部、弹窗）的 `--surface` 使用 `rgba(255, 255, 255, 0.92)`，即 92% 不透明白色。底下的网点从面板边缘隐约透出，产生纸张质感。弹窗（Modal）例外，使用 `#FFFFFF` 完全不透明，防止与页面内容重叠。

## 四、布局规则

### 4.1 通用布局骨架

| 角色 | 布局结构 | 说明 |
|:---|:---|:---|
| 系统管理员 | 固定顶部栏 + 固定左侧栏 + 可滚动内容区 | 侧栏宽度 `240px` |
| 宿管 | 固定顶部栏 + 固定左侧栏 + 可滚动内容区 | 侧栏宽度 `220px` |
| 学生 | 固定顶部栏 + 水平标签导航 + 全宽居中内容 | 无侧栏，内容最大宽度 `960px` |

### 4.2 顶部栏（Header）

| 属性 | 值 |
|:---|:---|
| 高度 | `56px` |
| 背景 | `var(--surface)` |
| 底部边框 | `1px solid var(--border)` |
| z-index | `100` |
| 布局 | `display: flex; justify-content: space-between; align-items: center` |
| 水平内边距 | `24px` |

顶部栏必须固定在视口顶部（`position: fixed; top: 0`）。

### 4.3 侧栏（Sidebar）

| 属性 | 值 |
|:---|:---|
| 位置 | 固定在顶部栏下方左侧 |
| 背景 | `var(--surface)` |
| 右边框 | `1px solid var(--border)` |
| z-index | `50` |
| 垂直内边距 | `16px 0` |
| 超出内容 | `overflow-y: auto` |

侧栏导航项分组显示，每组有等宽小字眉标标题。当前页激活项：左侧 `2px` 错误色边框 + `var(--accent-soft)` 背景 + 错误色文字。待处理数量用红色徽标显示在导航项右侧。

### 4.4 响应式断点

| 断点 | 布局变化 |
|:---|:---|
| `≤1200px` | 统计卡片 4→2 列，功能模块 3→2 列，筛选区纵向排列 |
| `≤1024px` | 统计卡片 3→2 列，快捷操作 4→2 列 |
| `≤768px` | 侧栏隐藏，所有网格→1 列，表单单列，标签栏横向滚动 |

禁止出现水平滚动条（`overflow-x: hidden` 或合理布局）。

## 五、组件规范

### 5.1 卡片（Card）

**用途：** 承载独立信息块（统计数据、功能入口、用户信息）。

**基础样式：**
- 背景：`var(--surface)`
- 边框：`1px solid var(--border)`
- 圆角：`var(--radius-lg)`（`12px`）
- 内边距：`24px`（上下左右）
- 过渡：`border-color 0.15s, box-shadow 0.2s, transform 0.2s`

**悬浮效果：**
- 边框变为 `color-mix(in oklch, var(--accent) 30%, transparent)`
- 阴影：`0 8px 24px rgba(0, 0, 0, 0.08)`
- 上移：`transform: translateY(-3px)`

**禁止：** 卡片左侧彩色边框装饰、卡片内渐变背景、卡片间投影堆叠。

### 5.2 按钮（Button）

#### 5.2.1 深色按钮（`.btn-primary`）

**用途：** 页面主操作（新增、提交、确认）。每个屏幕最多 1 个。

| 属性 | 默认态 | 悬浮态 |
|:---|:---|:---|
| 背景 | `var(--accent)` | `color-mix(in oklch, var(--accent) 88%, black)` |
| 文字 | `var(--surface)` | `var(--surface)` |
| 边框 | `var(--accent)` | `var(--accent)` |
| 变换 | 无 | `scale(1.03)` |
| 过渡 | `all 0.15s` | — |

#### 5.2.2 浅色/边框按钮（`.btn-secondary`）

**用途：** 次要操作（取消、返回、重置）。可多个。

| 属性 | 默认态 | 悬浮态 |
|:---|:---|:---|
| 背景 | `transparent` | `var(--hover-bg)`（`rgb(245, 244, 237)`） |
| 文字 | `var(--fg)` | `var(--accent)` |
| 边框 | `var(--border)` | `var(--accent)` |
| 变换 | 无 | `scale(1.03)` |
| 过渡 | `all 0.15s` | — |

#### 5.2.3 幽灵按钮（`.btn-ghost`）

**用途：** 表格行内操作（编辑、查看详情）、轻量链接式操作。

| 属性 | 默认态 | 悬浮态 |
|:---|:---|:---|
| 背景 | `transparent` | `var(--hover-bg)` |
| 文字 | `var(--muted)` | `var(--accent)` |
| 边框 | `transparent` | `transparent` |
| padding | `6px 10px` | 不变 |

#### 5.2.4 小号按钮（`.btn-sm`）

在任何按钮类型上追加 `.btn-sm`：
- padding：`5px 12px`
- 字号：`12px`
- 内含 SVG 尺寸：`14px × 14px`

#### 5.2.5 按钮尺寸枚举

| 尺寸 | padding | 字号 | 用途 |
|:---|:---|:---|:---|
| 默认 | `9px 18px` | `14px` | 页面级操作按钮 |
| 小号 | `5px 12px` | `12px` | 表格行内操作、筛选区按钮 |

禁止出现 `32px` 高度以上的巨型按钮。

### 5.3 表格（Table）

#### 5.3.1 基础结构

- 容器：`.data-panel`，背景 `var(--surface)`，边框 `1px solid var(--border)`，圆角 `var(--radius-lg)`
- 基础 `table`：`border-collapse: separate; border-spacing: 0`
- 表头 th：背景 `var(--bg)`，字号 `11px`，等宽字体，`text-transform: uppercase`，`letter-spacing: 0.04em`，颜色 `var(--muted)`，底边框 `1px solid var(--border)`
- 表格 td：padding `12px 16px`，`vertical-align: middle`

#### 5.3.2 列间竖线

列与列之间使用 `::after` 伪元素绘制竖线，颜色 `var(--col-line)`（`rgb(240, 238, 230)`）。

| 位置 | 竖线上下留白 | 说明 |
|:---|:---|:---|
| 表头 th | `top: 25%; bottom: 25%` | 竖线较短，居中 |
| 表体 td | `top: 15%; bottom: 15%` | 竖线较长，接近行高 |
| 最后一列 | 不绘制 | `th:not(:last-child)::after` / `td:not(:last-child)::after` |

竖线在行与行之间断开，不连通。实现方式：`border-collapse: separate` + 行底边框 + `::after` 伪元素。

#### 5.3.3 行样式

| 状态 | 样式 |
|:---|:---|
| 默认 | 无特殊背景 |
| 奇数行 | `background: var(--stripe)`（`rgba(0, 0, 0, 0.025)`） |
| 悬浮行 | `background: var(--hover-bg)`（`rgb(245, 244, 237)`） |
| 行底边框 | `border-bottom: 1px solid var(--border)` |
| 最后一行 | 无底边框 |

#### 5.3.4 数据格式

| 数据类型 | 格式 | 字体 |
|:---|:---|:---|
| 编号/学号/工号 | 等宽显示，不截断 | `var(--font-mono)` |
| 日期时间 | `YYYY-MM-DD HH:mm` | `var(--font-mono)` |
| 电话号码 | 脱敏显示，如 `138****5678` | `var(--font-mono)` |
| 身份证号 | 脱敏显示，如 `110101********1234` | `var(--font-mono)` |
| 状态标签 | pill 样式 | `var(--font-mono)` |

### 5.4 状态标签（Pill / Badge）

**用途：** 在表格、列表中标记状态分类。

| 类型 | 背景色 | 文字色 | 用途 |
|:---|:---|:---|:---|
| `pill-admin` | `color-mix(in oklch, var(--accent) 12%, transparent)` | `var(--accent)` | 管理员角色 |
| `pill-manager` | `var(--accent-2-soft)` | `var(--accent-2)` | 宿管角色 |
| `pill-student` | `var(--fg-soft)` | `var(--muted)` | 学生角色 |
| `pill-danger` | `color-mix(in oklch, var(--accent) 12%, transparent)` | `var(--accent)` | 已超时、紧急 |
| `pill-pending` | `color-mix(in oklch, var(--accent-2) 14%, transparent)` | `rgb(154, 93, 46)` | 待处理、待审批 |
| `pill-processing` | `color-mix(in oklch, var(--status-ok) 14%, transparent)` | `var(--status-ok)` | 处理中 |
| `pill-active` | `color-mix(in oklch, var(--status-ok) 10%, transparent)` | `var(--status-ok)` | 空余床位、在线 |
| `pill-quiet` / `pill-done` | `var(--fg-soft)` | `var(--muted)` | 满员、已完成、禁用 |

**Pill 基础样式：**
- padding：`3px 10px`
- 圆角：`999px`
- 字体：`var(--font-mono)`
- 字号：`11px`
- 字重：500
- 字间距：`0.02em`

### 5.6 用户下拉菜单（User Dropdown）

**结构：** 头像 + 姓名 + 箭头 SVG，hover 展开下拉面板。

**触发器（`.user-trigger`）：**
- padding：`4px 8px 4px 4px`
- 圆角：`var(--radius)`
- 悬浮底色：`var(--hover-bg)`
- 过渡：`background 0.12s`

**箭头（`.chevron`）：**
- 尺寸：`14px × 14px`
- 默认朝下（`v` 形），hover 时 `transform: rotate(180deg)` 逆时针旋转 180° 变为朝上（`^` 形）
- 过渡：`transform 0.2s`

**下拉面板（`.user-menu`）：**
- 背景：`var(--surface)`
- 边框：`1px solid var(--border)`
- 圆角：`var(--radius)`
- 阴影：`0 8px 24px rgba(0, 0, 0, 0.08)`
- 内边距：`4px`
- 默认状态：`opacity: 0; visibility: hidden; transform: translateY(-4px)`
- 展开状态：`opacity: 1; visibility: visible; transform: translateY(0)`
- 过渡：`opacity 0.15s, transform 0.15s, visibility 0.15s`
- z-index：`200`

**菜单项（`.dropdown-item`）：**
- padding：`8px 12px`
- 圆角：`6px`（`var(--radius) - 2px`）
- 悬浮底色：`var(--hover-bg)`
- 过渡：`background 0.1s`

**菜单分隔线（`.dropdown-divider`）：**
- 高度：`1px`
- 背景：`var(--border)`
- 上下间距：`4px`

### 5.7 自定义下拉选择框（Custom Select）

**替代原生 `<select>`，保持风格统一。**

**触发器（`.cselect-trigger`）：**
- 与输入框同高、同边框、同圆角、同背景
- padding：`7px 12px`（筛选区）/ `9px 12px`（表单区）
- 背景：`var(--bg)`
- 边框：`1px solid var(--border)`，圆角 `var(--radius)`
- 悬浮态：边框变为 `var(--accent)`
- 展开态：边框变为 `var(--accent)`

**箭头（`.cselect-arrow`）：**
- 尺寸：`14px × 14px`
- 默认朝下，展开时 `transform: rotate(180deg)` 顺时针旋转 180° 变为朝上
- 过渡：`transform 0.2s`

**下拉面板（`.cselect-panel`）：**
- 背景：`#FFFFFF`（完全不透明）
- 边框：`1px solid var(--border)`
- 圆角：`var(--radius)`
- 阴影：`0 8px 24px rgba(0, 0, 0, 0.08)`
- 最大高度：`200px`，超出滚动
- 默认/展开过渡与 `.user-menu` 一致

**选项（`.cselect-option`）：**
- padding：`7px 12px`
- 圆角：`6px`
- 悬浮底色：`var(--hover-bg)`
- 选中态：背景 `var(--accent-soft)`，文字 `var(--accent)`，字重 500

**交互规则：**
- 点击触发器切换面板展开/收起
- 点击选项后自动收起面板，更新显示文本
- 点击页面任意区域收起所有展开的下拉框

## 七、交互逻辑规范

### 7.1 何时使用深色按钮 vs 浅色按钮

- **深色按钮（`.btn-primary`）：** 每个功能区域的唯一主操作。如"新增用户"、"确认提交"、"办理入住"。每个屏幕可见区域内最多 1 个。
- **浅色按钮（`.btn-secondary`）：** 次要操作或否定性操作。如"取消"、"重置"、"返回"。可多个并列。
- **幽灵按钮（`.btn-ghost`）：** 表格行内操作、导航链接式操作。如"编辑"、"查看"、"处理"。每行可多个。

### 7.2 弹窗透明度规则

弹窗（Modal）背景**必须使用 `#FFFFFF` 完全不透明**，不得使用 `var(--surface)`（`rgba(255, 255, 255, 0.92)`）。原因：弹窗覆盖在页面内容上方，半透明背景会导致弹窗内容与底层页面内容视觉重叠，降低可读性。

### 7.3 表格列竖线断开规则

列间竖线**必须在行与行之间断开**，不得从表头到表尾贯通。实现方式：`border-collapse: separate` + 每行独立 `border-bottom` + `td::after` 伪元素绘制竖线（带 `top/bottom` 留白）。竖线颜色 `rgb(240, 238, 230)`，比行底边框色 `rgb(235, 230, 223)` 更浅，形成层级区分。

### 7.4 空状态处理

表格无数据时，不得显示空白表格骨架。必须显示居中空状态提示：
- 图标：灰色描线 SVG，`32px × 32px`，颜色 `var(--border)`
- 文字：`暂无数据`，颜色 `var(--muted)`，字号 `14px`
- 容器：与数据表格相同的面板样式（背景、边框、圆角）

### 7.5 删除操作双确认

删除操作必须执行双重确认：
1. 第一次：点击删除按钮后弹出确认弹窗，显示待删除项名称
2. 第二次：弹窗中点击"确认删除"按钮执行操作
3. 禁止单击直接删除

### 7.6 加载状态

按钮提交操作时必须显示加载态：
- 按钮文字替换为加载图标（旋转 spinner）
- 按钮设为 `disabled` 状态
- 操作完成后恢复

## 九、文件命名与引用规范

### 9.1 CSS 令牌位置

所有设计令牌（颜色、字体、间距、圆角）**必须**定义在 `<style>` 块顶部的 `:root` 选择器内。文件其他位置禁止出现 raw hex（`#xxx`）或 raw rgb 值——必须引用变量。

### 9.2 页面间一致性

所有同角色页面（如管理员的所有页面）**必须**使用完全相同的 `:root` 令牌块和布局骨架（header + sidebar + main）。不允许不同页面使用不同色调或不同侧栏宽度。

### 9.3 文件结构

| 文件类型 | 命名规则 | 示例 |
|:---|:---|:---|
| 角色主页 | `{role}-dashboard.html` | `admin-dashboard.html` |
| 角色子页 | `{role}-{module}.html` | `admin-user.html` |
| CSS | 内联于 `<style>` | — |
| JS | 内联于 `<script>` | — |
