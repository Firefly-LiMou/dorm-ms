# 高校公寓管理系统 后端架构设计文档V2

## 一、设计目标

### 1.1 量化指标
- 单接口响应时间控制在1秒以内
- 支持50人同时在线访问
- 核心业务方法测试覆盖率达到80%以上

### 1.2 约束条件
- JDK版本：11
- 技术栈：Spring 5.3.x + Spring MVC + MyBatis 3.5.x + Maven + Tomcat 9.x + MySQL 8.0
- 架构模式：单体三层架构（Controller → Service → DAO）
- 部署方式：WAR包部署至Tomcat

## 三、整体分层架构

### 3.1 架构总览

```
┌───────────────────────────────────────────────────┐
│                   视图层 (View)                    │
│  JSP + Bootstrap + jQuery                         │
└─────────────────┬─────────────────────────────────┘
                  │ HTTP请求/响应
┌─────────────────▼─────────────────────────────────┐
│                 表现层 (Controller)                │
│  Spring MVC控制层：请求接收、参数校验、结果封装    │
└─────────────────┬─────────────────────────────────┘
                  │ 方法调用
┌─────────────────▼─────────────────────────────────┐
│                  业务层 (Service)                  │
│  业务逻辑处理、事务控制、状态流转、多表联动        │
└─────────────────┬─────────────────────────────────┘
                  │ 数据访问
┌─────────────────▼─────────────────────────────────┐
│                 持久层 (DAO/Mapper)                │
│  MyBatis Mapper接口 + XML：SQL映射、CRUD操作       │
└─────────────────┬─────────────────────────────────┘
                  │ JDBC连接
┌─────────────────▼─────────────────────────────────┐
│                  数据层 (Database)                 │
│  MySQL 8.0：业务数据持久化存储                     │
└───────────────────────────────────────────────────┘

【横切公共组件（Spring AOP / 拦截器）】
┌───────────────────────────────────────────────────┐
│  权限拦截器、操作日志切面、全局异常处理、事务管理  │
└───────────────────────────────────────────────────┘
```

### 3.2 各层职责划分

| 层级 | 核心职责 | 技术实现 | 约束 |
|------|----------|----------|------|
| Controller | 接收请求、参数校验、调用Service、封装返回 | @Controller、@ResponseBody、@Valid | 禁止编写业务逻辑、禁止直接操作数据库 |
| Service | 业务逻辑、事务控制、业务参数校验 | @Service、@Transactional | 禁止处理HTTP请求、禁止拼接SQL |
| DAO | 数据库CRUD、SQL映射 | Mapper接口 + XML | 禁止编写业务逻辑、SQL必须预编译 |
| 横切组件 | 登录校验、权限拦截、操作日志、异常处理 | AOP、拦截器、@ControllerAdvice | 与业务代码解耦 |

## 五、公共组件设计

### 5.1 统一返回结果

```java
public class Result<T> {
    private Integer code;           // 状态码
    private String msg;             // 提示信息
```

**状态码规范**：
- 200：操作成功
- 400：参数错误
- 401：未登录
- 403：无权限
- 500：系统内部错误

### 5.3 枚举定义

| 枚举类 | 对应字段 | 枚举值 |

**使用规范**：
- 所有枚举统一放在`com.huuc.dormitory.common.enums`包下
- 每个枚举包含`code`（编码）、`desc`（描述）两个属性
- 提供`getByCode()`静态方法
- 业务代码中禁止直接使用数字字面量，必须通过枚举引用

### 5.5 操作日志（课程亮点）

**实现方式**：自定义@OperLog注解 + AOP切面

```java
// 注解定义
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface OperLog {
    String module();           // 操作模块
    OperTypeEnum type();       // 操作类型
    String desc() default "";  // 操作描述
}

// 切面实现
@Aspect
@Component
public class OperLogAspect {
    // 拦截@OperLog注解方法
    // 获取操作人、IP、参数
    // 异步写入sys_oper_log表
}
```

**使用示例**：
```java
@OperLog(module = "楼栋管理", type = OperTypeEnum.ADD, desc = "新增楼栋")
@PostMapping("/add")
public Result<Void> addBuilding(@RequestBody BuildingDTO dto) { ... }
```

## 六、用户认证与权限设计

### 6.1 认证流程

```
用户提交账号密码 → Controller接收 → Service校验
    → 验证用户名存在
    → 验证密码正确（MD5加密比对）
    → 验证账号状态正常
    → 将用户信息存入Session
    → 返回登录成功
```

### 6.2 权限拦截

**登录拦截器**：校验Session中是否存在用户信息，未登录则重定向至登录页

**权限拦截器**：使用AntPathMatcher精确路径匹配
- /admin/** 仅管理员可访问
- /dorm/** 仅宿管可访问
- /student/** 仅学生可访问

### 6.3 首次登录强制修改密码

**实现方式**：比对密码是否等于默认密码（年级+手机号的MD5值）

**登录接口返回**：
```json
{
  "code": 200,
  "data": { "userId": 1, "username": "admin", "roleType": 1 },
  "extra": { "needChangePassword": true }
}
```

## 七、数据传输对象设计

### 7.1 DTO使用原则

- DTO用于接收前端请求参数
- 简单场景直接使用@Valid注解校验
- 复杂场景在Service层进行业务参数校验

### 7.2 Lombok使用规范

**允许使用**：
- @Getter / @Setter：实体类、DTO、VO
- @ToString：实体类、DTO
- @NoArgsConstructor：实体类

**禁止使用**：
- @Data（可能引入equals/hashCode问题）
- @Builder（增加复杂度）
- @AllArgsConstructor（不利于扩展）

## 八、MyBatis持久层设计

### 8.1 开发模式

- Mapper接口 + XML映射文件
- 接口与XML同包同名自动映射
- 所有参数使用`#{}`预编译，杜绝SQL注入

### 8.2 SQL规范

- 条件查询使用`<if>`、`<where>`动态SQL标签
- 批量操作使用`<foreach>`标签
- 多表关联查询使用`resultMap`显式映射
- 新增操作使用`useGeneratedKeys="true"`回填主键

### 8.3 分页实现

使用PageHelper插件：
```java
PageHelper.startPage(pageNum, pageSize);
List<User> users = mapper.selectList(query);
PageInfo<User> pageInfo = new PageInfo<>(users);
```

## 九、声明式事务管理

### 9.1 配置方式

在spring-service.xml中配置事务管理器和注解驱动。

### 9.2 使用规范

- 核心写方法添加@Transactional注解
- 默认遇到RuntimeException回滚
- 纯查询方法设置readOnly = true

### 9.3 事务覆盖范围

- 入住办理、退宿办理
- 调宿审批通过
- 批量删除操作
- 涉及多表写操作的业务

## 十、安全设计

### 10.1 数据安全

- 密码MD5加密存储，禁止明文
- 敏感数据脱敏输出（身份证、手机号）
- 核心业务表禁止物理删除

### 10.2 访问安全

- 所有业务接口必须经过登录拦截器校验
- 基于角色的接口级权限校验
- 学生查询个人数据强制从Session获取用户ID

### 10.3 输入安全

- MyBatis全部使用预编译语句
- 所有入参做合法性校验
- 使用Hibernate Validator注解校验

## 十一、请求处理全流程

```
1. 客户端发起HTTP请求
2. Tomcat接收，转发至DispatcherServlet
3. 执行拦截器链：登录校验 → 权限校验
4. 进入Controller层，参数绑定、校验
5. 调用Service层，执行业务逻辑
6. Service调用DAO层，访问数据库
7. 返回结果链：DAO → Service → Controller
8. Controller封装Result返回
9. AOP切面记录操作日志（如适用）
10. 全局异常处理器捕获异常（如发生）
```

## 十二、模块职责与边界

### 12.1 用户认证与权限模块

**职责**：登录/登出、权限拦截、个人信息维护、用户管理

**接口**：
- POST /login - 用户登录
- POST /logout - 用户登出
- GET /currentUser - 获取当前用户
- GET /needChangePassword - 判断是否需要修改密码

### 12.2 宿舍基础信息模块

**职责**：楼栋、房间、床位的CRUD管理

**约束**：删除上级数据前必须校验下级是否存在关联数据

### 12.3 入住与调宿模块

**职责**：入住登记、退宿办理、调宿申请审批

**约束**：所有操作必须在事务中执行，保证数据一致性

### 12.4 日常运维模块

**职责**：报修管理、晚归登记、访客登记

**约束**：按角色隔离数据（学生看本人、宿管看本楼栋、管理员看全部）

### 12.5 操作日志模块

**职责**：自动记录管理端增删改操作

**约束**：日志记录失败不影响主业务流程

## 十四、验收标准

### 14.1 分层架构验收

- 严格遵循MVC三层架构，无跨层调用
- 公共组件与业务代码解耦

### 14.2 技术栈落地验收

- Spring MVC请求路由、参数绑定、视图解析功能正常
- MyBatis SQL编写规范，预编译、动态SQL正确应用
- Spring AOP操作日志切面无侵入实现
- 声明式事务在核心业务场景生效

### 14.3 安全与质量验收

- 登录认证、角色权限隔离生效
- 密码加密存储，无SQL注入、XSS风险
- 统一返回、全局异常处理生效
