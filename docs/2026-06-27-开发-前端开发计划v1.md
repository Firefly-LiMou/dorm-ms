# 高校公寓管理系统 前端开发计划V1

## 一、设计目标

### 1.1 量化指标
- 页面首屏加载时间控制在2秒以内
- AJAX请求响应处理时间控制在500毫秒以内
- 支持50人同时在线访问，无页面卡顿
- 前端代码覆盖率（关键交互逻辑）达到80%以上

### 1.2 约束条件
- 前端技术栈：JSP + Bootstrap 5 + jQuery 3.6
- 浏览器兼容：Chrome 90+、Firefox 88+、Edge 90+
- 字符编码：UTF-8
- 认证方式：Session（Cookie）
- 视图解析器配置：前缀`/WEB-INF/jsp/`，后缀`.jsp`
- 静态资源路径：`/static/`

## 三、整体架构设计

### 3.1 架构总览

```
┌───────────────────────────────────────────────────┐
│                   浏览器端 (Browser)               │
│  HTML + CSS + JavaScript                          │
└─────────────────┬─────────────────────────────────┘
                  │ HTTP请求
┌─────────────────▼─────────────────────────────────┐
│                 视图层 (View)                      │
│  JSP页面：数据展示、表单提交、交互处理              │
│  按角色分包：admin/ dorm/ student/                │
└─────────────────┬─────────────────────────────────┘
                  │ AJAX请求
┌─────────────────▼─────────────────────────────────┐
│               控制层 (Controller)                  │
│  Spring MVC：接收请求、返回JSON或视图              │
└─────────────────┬─────────────────────────────────┘
                  │ 方法调用
┌─────────────────▼─────────────────────────────────┐
│                业务层 (Service)                    │
│  业务逻辑处理、事务控制                            │
└───────────────────────────────────────────────────┘
```

### 3.2 前端职责划分

| 组件 | 核心职责 | 技术实现 | 约束 |
|------|----------|----------|------|
| JSP页面 | 数据展示、表单收集、页面跳转 | JSP + JSTL + Bootstrap | 禁止在JSP中编写Java业务逻辑 |
| JavaScript | 交互处理、AJAX请求、数据验证 | jQuery + 原生JS | 禁止直接操作DOM进行业务逻辑处理 |
| CSS | 样式定义、响应式布局 | Bootstrap + 自定义CSS | 禁止使用内联样式 |
| AJAX | 异步数据请求、JSON数据交换 | jQuery.ajax | 必须处理错误响应 |

**pagination.jsp** - 分页组件

- 成功、警告、错误、信息四种类型
- 自动消失（3秒）
- 参数：`type`、`message`

## 七、前端编码规范

### 7.1 JSP页面规范

**页面结构规范**
- 每个JSP页面必须声明`contentType="text/html;charset=UTF-8"`
- 必须使用`<!DOCTYPE html>`声明文档类型
- 必须包含`<meta charset="UTF-8">`字符编码声明
- 页面标题格式：`功能名称 - 高校公寓管理系统`

**JSP标签使用规范**
- 优先使用JSTL标签库，减少Java脚本代码
- 使用`<c:if>`、`<c:forEach>`等标签进行条件判断和循环
- 使用`${pageContext.request.contextPath}`获取项目路径，禁止硬编码路径

**禁止项**
- 禁止在JSP中编写Java业务逻辑代码
- 禁止使用`<% %>`脚本片段（除必要的变量声明外）

### 7.2 JavaScript编码规范

**命名规范**
- 变量名、函数名：小驼峰命名法（`userName`、`getUserInfo`）
- 常量：全大写+下划线分隔（`MAX_PAGE_SIZE`、`API_BASE_URL`）
- 类名、构造函数：大驼峰命名法（`UserManager`、`PageHelper`）
- jQuery对象变量：以`$`开头（`$form`、`$table`）

**代码格式规范**
- 统一使用4空格缩进，禁止使用Tab
- 每行代码长度不超过120个字符
- 字符串统一使用单引号（`'string'`）
- 语句结尾必须添加分号（`;`）
- 左大括号不换行，右大括号单独一行

**函数规范**
- 函数长度原则上不超过50行，超长函数必须拆分
- 函数必须添加JSDoc注释，说明功能、参数、返回值
- 禁止使用`arguments`对象，使用rest参数（`...args`）代替
- 回调函数使用箭头函数或提前定义的命名函数

**变量规范**
- 优先使用`const`声明常量，其次使用`let`声明变量
- 禁止使用`var`声明变量
- 变量声明必须在使用前完成
- 禁止声明未使用的变量

## 九、AJAX请求规范

### 9.1 请求封装

```javascript
/**
 * 统一AJAX请求封装
 * @param {string} url - 请求路径
 * @param {string} method - 请求方法（GET/POST）
 * @param {object} data - 请求数据
 * @param {function} successCallback - 成功回调
 * @param {function} errorCallback - 错误回调（可选）
 */
$.ajaxRequest = function(url, method, data, successCallback, errorCallback) {
    $.ajax({
        url: url,
        type: method,
        contentType: 'application/json',
        data: method === 'GET' ? data : JSON.stringify(data),
        dataType: 'json',
        success: function(result) {
            if (result.code === 200) {
                successCallback(result);
            } else if (result.code === 401) {
                // 未登录，跳转登录页
                window.location.href = '/dorm-ms/login';
            } else if (result.code === 403) {
                // 无权限，显示提示
                $.toast('error', '无权限访问');
            } else {
                // 其他错误，显示错误信息
                if (errorCallback) {
                    errorCallback(result);
                } else {
                    $.toast('error', result.msg);
                }
            }
        },
        error: function(xhr, status, error) {
            $.handleError(xhr, status, error);
        }
    });
};
```

### 9.2 错误处理规范

| 状态码 | 处理方式 |
|--------|----------|
| 200 | 正常处理，调用successCallback |
| 400 | 显示后端返回的错误信息 |
| 401 | 跳转登录页 |
| 403 | 显示无权限提示 |
| 404 | 显示"资源不存在" |
| 409 | 显示冲突信息（如"用户名已存在"） |
| 500 | 显示"系统繁忙，请稍后重试" |

### 9.3 请求拦截器

```javascript
// 全局AJAX错误处理
$(document).ajaxError(function(event, xhr, settings, thrownError) {
    if (xhr.status === 401) {
        window.location.href = '/dorm-ms/login';
    }
});
```

## 十二、数据流向设计

### 12.1 页面加载流程

```
1. 用户访问页面URL
2. DispatcherServlet接收请求
3. 执行LoginInterceptor：校验Session中是否存在用户
4. 执行PermissionInterceptor：校验路径与角色匹配
5. 进入Controller层：返回视图名称
6. ViewResolver解析视图：拼接前缀+视图名+后缀
7. JSP页面渲染：结合Model数据生成HTML
8. 响应返回浏览器
```

### 12.2 AJAX请求流程

```
1. 用户触发交互（点击按钮、提交表单等）
2. JavaScript收集表单数据
3. 前端验证（jQuery Validation）
4. 发送AJAX请求（$.ajaxRequest）
5. 后端处理请求，返回JSON结果
6. 前端处理响应：
   - 成功：更新页面数据、显示成功提示
   - 失败：显示错误信息
7. 页面状态更新（刷新列表、关闭模态框等）
```

### 12.3 分页查询流程

```
1. 页面加载，初始化分页参数（pageNum=1, pageSize=10）
2. 调用查询函数，发送AJAX请求
3. 后端执行分页查询，返回PageInfo对象
4. 前端渲染数据表格
5. 前端渲染分页组件
6. 用户点击页码，更新pageNum参数
7. 重新调用查询函数
```
