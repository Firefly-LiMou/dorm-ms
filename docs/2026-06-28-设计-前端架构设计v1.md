# 高校公寓管理系统 前端架构设计文档V1

## 一、设计目标

### 1.1 量化指标
- 页面首屏加载时间控制在2秒以内
- AJAX请求响应处理时间控制在500毫秒以内
- 所有列表页面必须覆盖四种状态：正常数据、加载中、空数据、请求错误

### 1.2 约束条件
- 前端技术栈：JSP + Bootstrap 5 + jQuery 3.6 + JSTL
- 认证方式：Session（Cookie）
- 视图解析器配置：前缀`/WEB-INF/jsp/`，后缀`.jsp`
- 静态资源路径：`/static/`
- 项目上下文路径：`/dorm-ms`
- 统一AJAX请求封装：`$.ajaxRequest(url, method, data, successCallback, errorCallback)`
- 统一返回格式：`{code, msg, data, extra}`

## 四、公共组件设计

### 4.1 页面布局框架

每个页面采用统一的布局结构：

```
main-container (flex布局)
├── sidebar (fixed, 220px)        # 侧边栏导航
└── content-wrapper (margin-left: 220px)
    ├── header                    # 顶部导航栏
    ├── content-body              # 页面主体内容
    │   ├── 页面标题
    │   ├── 查询区域（可选）
    │   └── 列表/表单/详情区域
    └── footer                    # 底部信息
```

### 4.2 header.jsp

**功能**：
- 显示当前登录用户姓名和角色
- 退出登录按钮
- 修改密码模态框（Bootstrap Modal）
- 首次登录时自动弹出修改密码模态框（不可关闭）

**修改密码模态框**：
- 输入新密码（6-20位）
- 确认密码（须与新密码一致）
- 前端校验后提交 `POST /updatePassword`
- 成功后清除needChangePassword标记，跳转登录页

### 4.3 sidebar.jsp

**功能**：
- 按角色动态渲染菜单（通过JSTL判断`sessionScope.loginUser.roleType`）
- 平铺布局：所有菜单项始终可见，分类用小号文字分隔
- 当前页面高亮（URL前缀匹配）

**三类角色菜单结构**：
- 管理员：系统管理、宿舍管理、入住管理、日常运维
- 宿管：宿舍信息（只读）、入住管理、日常运维
- 学生：个人中心、住宿管理、日常服务

### 4.4 footer.jsp

页脚组件，显示版权信息。

## 五、公共JS设计

### 5.1 common.js — 核心工具函数

| 函数 | 签名 | 说明 |
|------|------|------|
| `$.ajaxRequest` | `(url, method, data, success, error)` | 统一AJAX封装，自动序列化参数、统一错误码处理（401跳转登录、403提示无权限） |
| `$.toast` | `(type, message, duration)` | Toast提示：success（绿色）、error（红色）、warning（黄色）、info（蓝色），默认3秒自动消失，顶部居中 |
| `$.confirm` | `(message, onConfirm, onCancel, title)` | Bootstrap模态框二次确认弹窗 |
| `$.buildUrl` | `(url)` | 拼接项目上下文路径 |
| `$.formatDate` | `(dateStr, format)` | 日期格式化，默认 `yyyy-MM-dd HH:mm:ss` |

**错误码统一处理**：
- 401 → 提示"未登录"，1.5秒后跳转登录页
- 403 → 提示"无权限访问"
- 400 → 显示返回的msg
- 500 → 显示返回的msg

### 5.2 header.js — 导航栏逻辑

| 函数 | 说明 |
|------|------|
| `showChangePasswordModal()` | 显示修改密码模态框 |
| `changePassword()` | 提交修改密码请求，前端校验密码长度与一致性 |

**首次登录强制修改密码**：页面加载时检查 `needChangePasswordFlag`，若为true则自动弹出修改密码模态框，设置`backdrop: 'static'`和`keyboard: false`禁止关闭。

### 5.3 页面脚本引入顺序

```html
<!-- 1. jQuery -->
<script src="${pageContext.request.contextPath}/static/js/jquery.min.js"></script>
<!-- 2. Bootstrap JS -->
<script src="${pageContext.request.contextPath}/static/vendor/bootstrap/js/bootstrap.bundle.min.js"></script>
<!-- 3. 公共JS -->
<script src="${pageContext.request.contextPath}/static/js/common.js"></script>
<!-- 4. 导航栏JS -->
<script>window.needChangePasswordFlag = '${sessionScope.needChangePassword}';</script>
<script src="${pageContext.request.contextPath}/static/js/header.js"></script>
<!-- 5. 页面专属JS（内联） -->
<script>/* 页面逻辑 */</script>
```

## 七、请求规范

### 7.1 统一请求方式

除登录接口使用 `$.ajax` + FormData 外，所有业务请求统一使用 `$.ajaxRequest` 封装：

```javascript
// GET分页查询
$.ajaxRequest('/admin/user/page', 'GET', {pageNum: 1, pageSize: 10}, success, error);

// POST提交表单
$.ajaxRequest('/admin/user/add', 'POST', postData, success, error);

// POST修改状态（URL路径参数）
$.ajaxRequest('/admin/user/toggleStatus/' + userId, 'POST', null, success, error);

// 文件上传（使用$.ajax + FormData）
var formData = new FormData();
formData.append('file', file);
$.ajax({
    url: $.buildUrl('/admin/user/import'),
    type: 'POST',
    data: formData,
    processData: false,
    contentType: false,
    dataType: 'json',
    success: success,
    error: error
});
```

### 7.2 参数传递规范

- 空值不传递：查询条件为空时不添加到参数对象中
- 分页参数：统一使用 `pageNum`、`pageSize`
- 学号查询参数名：统一使用 `studentNo`
- 模糊查询参数：空值不传，避免null序列化为字符串"null"

### 7.3 文件上传规范

- 仅CSV文件（扩展名校验 `.csv`）
- 文件大小限制1MB
- 使用FormData + $.ajax（不能使用$.ajaxRequest）
- 后端返回 `ImportResultDTO`（含成功数、失败数、失败明细）

- RESTful接口已预留，迁移仅需替换前端

## 九、验收标准

- 页面布局统一（sidebar + header + content + footer）
- 列表页四状态覆盖：加载中、空数据、请求错误、正常数据
- 所有表单有前端校验
- 敏感操作有二次确认
- 修改密码模态框首次登录不可关闭
- 移动端侧边栏自动隐藏
- 无console.log遗留，无Java脚本块
