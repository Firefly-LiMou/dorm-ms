## 二、管理员端：「系统总览」

### 2.1 页面设计

**文件**：重写 `admin/index.jsp`

**布局**：

```
┌──────────────────────────────────────────────┐
│  系统总览                                      │
├──────────┬──────────┬──────────┬─────────────┤
│ 入住率%   │ 本月报修  │ 本月晚归  │ 本月访客     │  ← 指标卡片行（4个）
│ 环形图     │ 数字      │ 数字      │ 数字         │
├──────────┴──────────┼──────────┴─────────────┤
│ 楼栋入住率柱状图      │ 近6月业务趋势折线图      │  ← 图表行
│ (ECharts)           │ (ECharts)              │
├─────────────────────┴────────────────────────┤
│ 待办事项                │ 近期操作日志          │  ← 列表行
│ 待处理报修(N条)+超时     │ 最近10条              │
│ 待审批调宿(N条)          │                      │
└───────────────────────┴───────────────────────┘
```

### 2.2 数据来源

| 展示项 | 数据接口 | 说明 |
|------|----------|------|
| 入住率 | 新建聚合接口 | 已入住床位/总床位 |
| 本月报修量 | 同上 | count本月repair |
| 本月晚归人次 | 同上 | count本月late_return |
| 本月访客量 | 同上 | count本月visitor |
| 楼栋入住率柱状图 | 同上 | 每楼栋的名称+入住率 |
| 近6月趋势折线图 | 同上 | 4条线按月统计 |
| 待处理报修 | 同上 | 前5条+总数 |
| 超时报修 | 同上 | pending>24h计数 |
| 待审批调宿 | 同上 | count pending |
| 近期操作日志 | 同上 | 最近10条 |

**后端**：新增 `GET /admin/dashboard`，单个聚合接口返回所有数据，减少请求。

### 2.3 空数据处理

- 新系统无数据：卡片显示0，"暂无数据"占位提示
- 本月无报修：折线图该月数据点为0，正常渲染
- 无待办事项：显示"暂无待办事项"绿色提示

## 三、宿管端：「运营看板」

### 3.1 页面设计

**文件**：重写 `dorm/index.jsp`

**布局**：

```
┌──────────────────────────────────────────────┐
│  运营看板 — XX楼（多楼栋显示"共N栋"）           │
├──────────┬──────────┬──────────┬─────────────┤
│ 入住率     │ 空余床位   │ 维修中    │ 今日新增    │  ← 本楼栋卡片
├─────────────────────┬────────────────────────┤

│ 横向柱状图(ECharts)   │ 环形图(ECharts)         │
├─────────────────────┴────────────────────────┤
│ 待处理报修(前5条)       │ 待审批调宿(N条)        │  ← 待办行
│ 超时红色标识            │ 在访未离校(N人)        │
└──────────────────────┴───────────────────────┘
```

### 3.2 多楼栋处理

**问题**：宿管可能管理多个楼栋，运营看板如何展示？

| 方案 | 描述 | 复杂度 | 推荐 |
|------|------|--------|------|
| A: 聚合 | 所有管理楼栋数据汇总到一个看板，图表每柱代表一个楼栋 | 低 | **已确认** |
| B: 切换 | 顶部tab选择楼栋，分别查看 | 中 | 备选 |

**决策**：采用方案A，聚合展示。

### 3.3 数据来源

**后端**：新增 `GET /dorm/dashboard`，从Session获取宿管ID，按管理的楼栋聚合数据。

## 五、公告通知模块

### 5.1 数据库设计

**表名**：`sys_notice`

| 字段 | 类型 | 说明 |
|------|------|------|
| notice_id | BIGINT PK | 公告ID |
| title | VARCHAR(100) | 标题（必填） |
| content | TEXT | 内容（Markdown格式） |
| notice_type | TINYINT | 分类：1-通知 2-维修 3-活动 4-紧急 |
| visible_scope | TINYINT | 范围：1-全部 2-按楼栋 |
| building_id | BIGINT | 楼栋ID（scope=2时必填，NULL=全部楼栋） |
| status | TINYINT | 状态：1-发布 2-置顶 3-下架 |
| publisher_id | BIGINT | 发布人ID |
| publish_time | DATETIME | 发布时间 |
| create_time | DATETIME | 创建时间 |
| update_time | DATETIME | 更新时间 |

### 5.2 权限矩阵

| 角色 | 发布 | 编辑/置顶/下架 | 查看范围 |
|------|------|---------------|----------|
| 管理员 | 可 | 可 | 全部公告 |
| 宿管 | 不可 | 不可 | 全局公告 + 本楼栋公告 |
| 学生 | 不可 | 不可 | 全局公告 + 本楼栋公告 |

### 5.3 API设计（5个）

| 接口 | 方法 | 说明 |
|------|------|------|
| `/admin/notice/page` | GET | 分页查询（管理员查全部） |
| `/admin/notice/add` | POST | 新增公告 |
| `/admin/notice/update` | POST | 编辑公告 |
| `/admin/notice/toggleStatus/{id}` | POST | 置顶/下架切换 |
| `/common/notice/visible` | GET | 当前用户可见公告（宿管/学生用） |

**路径决策**：放在 CommonController 下，对所有已登录用户自动开放。

### 5.4 页面设计（3页）

| 页面 | 路径 | 说明 |
|------|------|------|
| 公告管理列表 | `admin/notice/list.jsp` | 表格+新增/编辑/置顶/下架按钮 |
| 公告编辑页 | `admin/notice/edit.jsp` | 新增和编辑共用，Markdown编辑区+预览 |
| 公告详情弹窗 | 公共（Modal） | 首页点击公告卡片弹出，渲染Markdown |

**侧边栏位置**：管理员菜单"系统管理"下新增"公告管理"。

### 5.5 Markdown支持

| 方面 | 方案 |
|------|------|
| 编辑器 | **SimpleMDE**（轻量Markdown编辑器，~80KB） |
| 集成方式 | 本地化到 `static/vendor/simplemde/`，textarea 一行初始化 |
| 后端改动 | 零——表单提交的仍是原始Markdown文本 |
| 预览 | 内置实时预览窗格，工具栏含加粗/标题/列表/链接 |
| 渲染 | SimpleMDE 内置 marked.js 渲染，无需额外引入 |
| 安全 | 渲染前 XSS 过滤 |

**实现复杂度**：极低。本质是 textarea 的零配置替换，`new SimpleMDE({ element: textarea })` 一行代码完成。

### 5.6 首页公告展示

| 首页 | 展示方式 | 数据 |
|------|----------|------|
| 管理员系统总览 | 底部列表，标题+时间，最近3条 | `/admin/notice/page` |
| 宿管运营看板 | 卡片，最近3条 | `/common/notice/visible` |
| 学生我的主页 | 卡片，最近3条 | `/common/notice/visible` |

- 置顶公告优先显示，带红色"置顶"标签
- 空状态：卡片区显示"暂无公告通知"

## 六、可选精简：宿管房间床位合并

### 6.1 当前状态

宿管菜单下含3个只读页面：楼栋信息、房间信息、床位信息。

### 6.2 合并方案

| 合并前 | 合并后 |
|------|------|
| `dorm/building/list.jsp` | **保留**：楼栋信息只读表格 |
| `dorm/room/list.jsp` | **合并为** `dorm/room/list.jsp`：选楼栋→选房间→下方直接显示该房间床位 |
| `dorm/bed/list.jsp` | **删除** |

**改动**：房间页面增加床位展示区域，复用 `/dorm/bed/room/{roomId}` 接口。

### 6.3 侧边栏调整

```
宿舍信息
├── 楼栋信息    ← 保留
└── 房间床位    ← 合并（原"房间信息"+原"床位信息"）
```

减少1个页面文件和1个侧边栏菜单项。
