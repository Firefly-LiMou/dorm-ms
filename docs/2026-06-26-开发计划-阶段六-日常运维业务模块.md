# 阶段六：日常运维业务模块开发计划

## 一、模块概述

### 1.1 功能范围
- 报修管理：学生提交报修、宿管处理报修、报修查询
- 晚归登记：宿管录入晚归、晚归记录查询、基础统计
- 访客登记：宿管录入访客、访客记录查询、离校确认

### 1.2 角色权限矩阵
| 功能 | 管理员 | 宿管 | 学生 |
|------|--------|------|------|
| 报修提交 | - | - | ✓ |
| 报修处理 | ✓ | ✓（本楼栋） | - |
| 报修列表 | 全部 | 本楼栋 | 本人 |
| 晚归录入 | ✓ | ✓（本楼栋） | - |
| 晚归列表 | 全部 | 本楼栋 | 本人 |
| 晚归统计 | 全部楼栋 | 本楼栋 | - |
| 访客录入 | ✓ | ✓（本楼栋） | - |
| 访客离开确认 | ✓ | ✓（本楼栋） | - |
| 访客列表 | 全部 | 本楼栋 | - |

## 二、报修管理子模块

### 2.1 DormRepair（报修实体）

```java
@Getter
@Setter
@NoArgsConstructor
@ToString
public class DormRepair {
    private Long repairId;          // 报修单ID
    private Long studentId;         // 报修学生ID
    private Long roomId;            // 报修房间ID
    private Integer repairType;     // 报修类型
    private String repairContent;   // 报修详情描述
    private String contactPhone;    // 联系电话
    private LocalDateTime submitTime;    // 提交时间
    private Integer repairStatus;   // 处理状态：0-待处理 1-处理中 2-已完成
    private Long handlerId;         // 处理人ID
    private String handleResult;    // 处理结果
    private LocalDateTime finishTime;    // 处理完成时间
    private String remark;
    private LocalDateTime createTime;
    private LocalDateTime updateTime;
}
```

### 2.2 RepairDTO（报修提交）

```java
@Getter
@Setter
@NoArgsConstructor
@ToString
public class RepairDTO {
    @NotNull(message = "报修类型不能为空")
    private Integer repairType;

    @NotBlank(message = "报修内容不能为空")
    @Size(max = 500, message = "报修内容长度不能超过500个字符")
    private String repairContent;

    @NotBlank(message = "联系电话不能为空")
    private String contactPhone;
}
```

**说明**：不包含roomId，后端根据学生在住记录自动获取。

### 2.3 RepairHandleDTO（报修处理）

```java
@Getter
@Setter

public class RepairHandleDTO {
```

### 2.4 RepairVO（报修详情）

### 2.5 RepairService接口

```java
public interface RepairService {
    RepairVO getRepairById(Long repairId);
    PageInfo<RepairVO> getRepairList(DormRepair query, Integer pageNum, Integer pageSize);
    PageInfo<RepairVO> getMyRepairs(Long studentId, Integer pageNum, Integer pageSize);
    void submitRepair(RepairDTO dto, Long studentId);
    void handleRepair(Long repairId, Long handlerId);      // 接单处理
    void completeRepair(Long repairId, RepairHandleDTO dto, Long handlerId);  // 完结报修
}
```

**业务规则**：
- submitRepair：
  1. 校验学生当前有在住记录
  2. 自动获取学生在住的房间ID
  3. 插入报修记录，状态为「待处理」
- handleRepair：
  1. 校验报修记录存在且状态为「待处理」
  2. 更新状态为「处理中」，回填处理人ID
- completeRepair：
  1. 校验报修记录存在且状态为「处理中」
  2. 更新状态为「已完成」，回填处理结果、完成时间

### 2.6 RepairController

**管理员端**（/admin/repair）
- GET /list - 报修列表页面
- GET /page - 查询报修列表（分页）
- GET /{repairId} - 获取报修详情
- POST /handle/{repairId} - 接单处理
- POST /complete/{repairId} - 完结报修

**宿管端**（/dorm/repair）
- GET /list - 本楼栋报修列表页面
- GET /page - 查询本楼栋报修列表（分页）
- GET /{repairId} - 获取报修详情
- POST /handle/{repairId} - 接单处理
- POST /complete/{repairId} - 完结报修

**学生端**（/student/repair）
- GET /list - 我的报修列表页面
- GET /page - 查询我的报修列表（分页）
- GET /{repairId} - 获取报修详情
- POST /submit - 提交报修

### 4.4 VisitorService接口

```java
public interface VisitorService {
    VisitorVO getVisitorById(Long visitorId);
```

  1. 校验访客记录存在且未离开
  2. 更新离开时间为当前系统时间

## 七、异常处理

| 场景 | 异常信息 |
|------|----------|
| 学生无在住记录（报修） | "您当前无在住记录，无法提交报修" |
