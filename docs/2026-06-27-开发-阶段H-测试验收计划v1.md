## 三、步骤2：代码审查

### 2.1 后端代码审查

| 审查项 | 检查内容 | 依据 |
|--------|----------|------|
| 异常处理 | 无空catch块、无Exception兜底（全局处理器除外） | 编码规范§二 |
| 事务注解 | 核心写方法有@Transactional | 编码规范§二.3 |
| SQL预编译 | MyBatis全部使用#{},无${}拼接 | 编码规范§五.3 |
| 日志规范 | 无System.out.println、异常日志输出完整堆栈 | 编码规范§三 |
| 枚举使用 | 业务状态/类型无魔法数字 | 编码规范§四 |
| 分层规范 | Controller无业务逻辑、Service无HTTP操作 | 编码规范§五 |
| 越权防护 | 学生端从Session获取ID、宿管端按楼栋隔离 | 编码规范§六 |
| 密码安全 | 无明文存储/输出 | 编码规范§六 |

### 2.2 前端代码审查

| 审查项 | 检查内容 | 依据 |
|--------|----------|------|
| 硬编码 | 无角色类型/状态数字硬编码 | 前端规范§四.3 |
| 请求封装 | 使用$.ajaxRequest（login除外） | 前端规范§四.1 |
| 页面状态 | 列表页覆盖加载中/空数据/错误状态 | 前端规范§三.1 |
| 表单校验 | 所有表单有前端校验 | 前端规范§三.2 |
| 敏感操作 | 删除/审批有二次确认 | 前端规范§三.2 |
| console清理 | 无console.log/alert遗留 | 前端规范§六 |
| JSP规范 | 无Java脚本块（JSTL/EL替代） | 前端规范§二.2 |
| 外部依赖 | FontAwesome已本地化 | 前端规范§四 |

### 2.3 业务逻辑审查

| 审查项 | 检查内容 | 依据 |
|--------|----------|------|
| 入住流程 | 事务保证入住记录+床位状态原子更新 | 业务逻辑§三.1 |
| 调宿流程 | 审批通过后原床位释放+目标床位占用+新入住记录 | 业务逻辑§三.3 |
| 报修流程 | 房间自动获取、不依赖前端传入roomId | 业务逻辑§四.1 |
| 晚归流程 | 楼栋自动获取、不依赖前端传入buildingId | 业务逻辑§四.2 |
| 访客流程 | 身份证脱敏、离开时间自动填充 | 业务逻辑§四.3 |
| 密码规则 | 默认密码"huuc+手机号"、首次登录强制修改 | 业务逻辑§一 |

```

    - JSP + Bootstrap + jQuery + JSTL
```
