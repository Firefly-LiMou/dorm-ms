# 高校公寓管理系统 前后端接口文档V2

## 一、通用说明

### 1.1 基础信息

| 项目 | 说明 |
|------|------|
| 基础路径 | http://localhost:8080/dorm-ms |
| 数据格式 | JSON |
| 字符编码 | UTF-8 |
| 认证方式 | Session（Cookie） |

### 1.2 统一响应格式

```json
{
  "code": 200,
  "msg": "操作成功",
  "data": {},
  "extra": {}
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| code | Integer | 状态码 |
| msg | String | 提示信息 |
| data | Object | 返回数据 |
| extra | Object | 扩展数据（可选） |

### 1.3 状态码规范（V2更新）

| 状态码 | 说明 | 前端处理建议 |
|--------|------|--------------|
| 200 | 操作成功 | 正常处理 |
| 400 | 参数错误 | 显示错误信息 |
| 401 | 未登录/认证失败 | 跳转登录页 |
| 403 | 无权限 | 显示无权限提示 |
| 404 | 资源不存在 | 显示错误信息 |
| 409 | 状态冲突 | 显示错误信息 |
| 405 | 请求方法不支持 | 显示错误信息 |
| 500 | 系统内部错误 | 显示错误信息 |

### 1.4 分页请求参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| pageNum | Integer | 否 | 1 | 页码 |
| pageSize | Integer | 否 | 10 | 每页条数 |

### 1.5 分页响应格式

```json
{
  "code": 200,
  "msg": "操作成功",
  "data": {
    "pageNum": 1,
    "pageSize": 10,
    "total": 100,
    "pages": 10,
    "list": []
  }
}
```

### 1.6 拦截器响应说明（V2新增）

**未登录响应（AJAX请求）**：
```json
{
  "code": 401,
  "msg": "未登录",
  "data": null
}
```

**权限不足响应**：
```json
{
  "code": 403,
  "msg": "无权限访问",
  "data": null
}
```

## 二、认证模块

### 2.1 用户登录

**请求路径**：`/login`

**请求方法**：POST

**请求参数**：

| 参数 | 类型 | 必填 | 约束 | 说明 |
|------|------|------|------|------|
| username | String | 是 | 不能为空 | 登录账号 |
| password | String | 是 | 不能为空 | 登录密码 |

**请求示例**：
```json
{
  "username": "admin",
  "password": "123456"
}
```

**成功响应**：
```json
{
  "code": 200,
  "msg": "操作成功",
  "data": {
    "userId": 1,
    "username": "admin",
    "realName": "系统管理员",
    "roleType": 1,
    "gender": 1,
    "phone": "13800138000"
  },
  "extra": {
    "needChangePassword": false
  }
}
```

**响应字段说明**：

| 字段 | 类型 | 说明 |
|------|------|------|
| data.userId | Long | 用户ID |
| data.username | String | 登录账号 |
| data.realName | String | 真实姓名 |
| data.roleType | Integer | 角色类型：1-管理员 2-宿管 3-学生 |
| data.gender | Integer | 性别：1-男 2-女 |
| data.phone | String | 联系电话 |
| extra.needChangePassword | Boolean | 是否需要强制修改密码 |

**错误响应（V2更新）**：

| 错误码 | 错误信息 | 说明 |
|--------|----------|------|
| 401 | 用户名或密码错误 | 账号或密码不正确 |
| 401 | 账号已被禁用，请联系管理员 | 账号状态为禁用 |

### 2.5 修改密码

| 参数 | 类型 | 必填 | 约束 | 说明 |

```json
  "confirmPassword": "123456"
```

```json
{

  "msg": "操作成功",
  "data": null
```

| 错误码 | 错误信息 | 说明 |
|--------|----------|------|
| 400 | 两次输入的密码不一致 | 密码不一致 |
| 400 | 新密码不能为空 | 参数校验失败 |
| 400 | 密码长度必须在6-20个字符之间 | 参数校验失败 |

**说明**：所有角色通用，无需验证旧密码。修改后清除Session中的needChangePassword标记。

### 3.7 批量导入用户

**请求路径**：`/admin/user/import`

**请求方法**：POST

**Content-Type**：`multipart/form-data`

**请求参数**：

| 参数 | 类型 | 必填 | 约束 | 说明 |
|------|------|------|------|------|
| file | File | 是 | CSV格式，≤1MB | 用户数据文件 |

**CSV文件格式**（UTF-8 BOM编码）：
```
username,realName,roleType,gender,phone,grade,major,className
```

**校验规则**：

| 字段 | 规则 | 错误信息 |
|------|------|----------|
| username | 非空、不重复 | "用户名为空"或"用户名已存在" |
| realName | 非空 | "真实姓名为空" |
| roleType | 1/2/3 | "角色类型无效" |
| gender | 1/2 或留空 | "性别值无效" |
| phone | 11位数字 | "手机号格式错误" |
| grade | roleType=3时必填 | "学生年级为空" |
| major | roleType=3时必填 | "学生专业为空" |
| className | roleType=3时必填 | "学生班级为空" |

**默认密码**：`huuc + 手机号`，MD5加密存储。

**成功响应（全部成功）**：
```json
{
  "code": 200,
  "msg": "操作成功",
  "data": {
    "successCount": 10,
    "failCount": 0,
    "failDetails": []
  }
}
```

**成功响应（部分失败）**：
```json
{
  "code": 200,
  "msg": "操作成功",
  "data": {
    "successCount": 8,
    "failCount": 2,
    "failDetails": [
      {"rowNum": 3, "reason": "用户名已存在"},
      {"rowNum": 7, "reason": "手机号格式错误"}
    ]
  }
}
```

**错误响应**：

| 错误码 | 错误信息 | 说明 |
|--------|----------|------|
| 400 | 请选择要导入的文件 | 文件为空 |
| 400 | 仅支持CSV格式文件 | 文件扩展名不是.csv |

## 十七、V2版本更新说明

### 17.1 主要变更

1. **错误码细化**：从原来的统一500改为400/401/403/404/409/500，前端可根据错误码进行差异化处理
2. **拦截器响应格式**：权限校验失败和未登录场景返回JSON格式，而非HTML
3. **操作日志类型**：新增"登录"、"登出"操作类型

### 17.2 前端适配建议

1. **错误码处理**：
   - 400：显示后端返回的错误信息
   - 401：跳转登录页
   - 403：显示无权限提示
   - 404：显示"资源不存在"
   - 409：显示冲突信息（如"用户名已存在"）
   - 500：显示"系统繁忙，请稍后重试"

2. **拦截器响应**：
   - 未登录AJAX请求返回 `{"code":401,"msg":"未登录","data":null}`
   - 权限不足返回 `{"code":403,"msg":"无权限访问","data":null}`
