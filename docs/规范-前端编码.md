# 高校公寓管理系统 前端编码规范

适用范围：本项目前端代码（默认 JSP + Bootstrap + 原生 JavaScript，可选 Vue 2.x + Element UI）

约束对象：所有开发人员与代码生成 Agent

核心原则：贴合课程技术栈、交互统一、兼容后端接口、可维护、无冗余设计

## 一、代码风格规范

### 1.1 命名规范

- 文件命名：JSP 页面、静态资源文件统一小写 + 下划线分隔，按模块归集，如 `student_repair_list.jsp`
- HTML class、id：小写 + 中划线分隔，语义化命名，禁止拼音、无意义命名
- JS 变量、函数：小驼峰命名法，动词开头表述动作；单字符变量仅允许循环中使用 `i`、`j`
- JS 常量：全大写 + 下划线分隔，使用 `const` 声明
- 函数名必须表述动作，禁止用无意义缩写

### 1.2 代码格式

- HTML、JS 统一 4 空格缩进，禁止 Tab
- HTML 标签嵌套层级不超过 6 层，避免过深嵌套
- JS 单行长度不超过 120 字符，超长语句拆分
- JS 语句结尾统一加分号
- 左大括号不换行，右大括号单独一行，与 Java 规范保持一致

### 1.3 注释规范

- JSP 页面顶部添加功能注释，说明所属模块、适用角色
- 核心业务函数、复杂交互逻辑必须添加功能注释，说明入参、作用、注意事项
- 公共工具函数必须注释功能、入参、返回值
- 禁止提交大段注释掉的代码，禁止无意义注释

### 1.4 文件结构

- JSP 页面按角色 + 模块分目录：`/WEB-INF/jsp/admin/`、`/WEB-INF/jsp/student/`、`/WEB-INF/jsp/dorm/`
- 静态资源按类型分目录：`css`、`js`、`images`，公共资源与模块资源分离
- 公共头部、侧边栏、页脚抽取为独立片段，各页面统一引入，禁止重复代码

## 二、页面与组件规范

### 2.1 Bootstrap 使用规范

- 统一使用 Bootstrap 栅格系统做布局，禁止大量手写固定宽高
- 优先使用 Bootstrap 原生组件（表格、表单、按钮、模态框、分页），禁止重复造轮子
- 全系统按钮、表单、提示样式保持统一，禁止零散自定义样式
- 表格、表单宽度自适应，禁止硬编码固定像素宽度

### 2.2 JSP 开发规范

- JSP 仅负责数据展示与简单逻辑判断，核心业务逻辑全部交由后端处理
- 使用 JSTL 核心标签库实现循环、条件判断，禁止在页面内编写复杂 Java 业务代码
- 公共页面片段使用 `include` 指令引入，避免重复编码
- 路径统一使用项目上下文路径，禁止硬编码绝对路径

### 2.3 Vue 可选方案规范（仅启用 Vue 时生效）

- 单文件组件结构清晰：template、script、style 分层
- 组件名大驼峰命名，props 必须声明类型与默认值
- 统一使用 Element UI 组件，禁止混用大量自定义原生组件
- 数据驱动视图，禁止直接操作 DOM

## 三、异常与状态处理规范

### 3.1 页面状态全覆盖

- 所有数据列表页面必须覆盖四种状态：正常数据、加载中、空数据、请求错误
- 加载中：展示加载提示，禁止页面空白
- 空数据：展示明确空状态提示，禁止空表格无任何反馈
- 请求错误：展示错误提示与重试按钮，禁止静默失败

### 3.2 表单与交互处理

- 所有提交表单必须做前端基础校验：非空校验、长度校验、格式校验（手机号、学号等）
- 校验不通过时定位到对应字段，给出明确错误提示
- 提交过程中禁用提交按钮，防止重复提交
- 敏感操作（删除、审批）必须添加二次确认弹窗

### 3.3 边界与异常处理

- 所有接口请求必须捕获异常，统一错误提示，禁止请求失败无反馈
- 后端返回非成功状态码时，必须展示错误信息，禁止忽略错误
- 登录失效、权限不足场景统一跳转登录页或展示无权限提示
- 分页边界处理：第一页禁用上一页，最后一页禁用下一页
- 长文本溢出做省略处理，禁止破坏页面布局

## 四、接口与数据规范

### 4.1 请求规范

- 统一请求方式与参数格式，与后端约定对齐，禁止零散拼接请求代码
- 所有请求必须设置超时时间，避免长时间无响应
- 接口路径统一管理，禁止在业务代码中硬编码接口地址
- 分页查询统一传递 `pageNum`、`pageSize` 参数，与后端分页规范一致

### 4.2 返回值处理

- 统一解析后端 `Result` 结构，按 `code` 判断请求结果
- 日期、时间统一格式化展示，禁止直接展示原始时间戳
- 禁止直接依赖后端原始字段名，必须做兼容处理
- 状态、类型字段优先展示中文描述，禁止直接展示数字编码

### 4.3 硬编码治理

- 业务状态、类型禁止直接写死数字判断，统一封装字典映射
- 接口地址、公共配置统一抽取为常量，禁止散落在业务代码中
- 相同提示文案、校验规则统一抽取，禁止重复编写

## 五、安全编码规范

- XSS 防护：用户输入内容展示时做转义处理，禁止直接渲染原始 HTML 内容
- 敏感数据脱敏：身份证号、手机号展示时中间字符替换为 `*`，禁止明文展示
- 权限控制：前端菜单按角色动态展示；但禁止仅靠前端隐藏做权限管控，依赖后端二次校验
- 退出登录必须清除本地用户信息与缓存，禁止残留敏感数据
- 禁止在前端存储密码、密钥等敏感信息

## 六、调试与日志规范

- 开发阶段使用 `console` 输出调试信息，禁止使用弹窗调试
- 交付版本必须清理所有 `console.log`、`alert` 调试语句
- 错误场景必须输出错误详情，便于问题排查
- 禁止在控制台输出密码、身份证等敏感信息

## 七、验收检查清单

出现以下任意一项，代码审查不通过：

1. 列表页面缺少加载中、空数据、错误状态任意一种，异常场景无提示
2. 表单无前端校验，提交无防重复处理
3. 接口请求无异常捕获，请求失败页面无反馈
4. 敏感信息明文展示，未做脱敏处理
5. 代码中散落大量硬编码接口路径、硬编码状态数字
6. 公共组件、公共样式未抽取，存在大量重复代码
7. JSP 页面中编写复杂 Java 业务逻辑，职责越界
8. 遗留注释掉的代码、调试用 `console.log`、`alert`
9. 命名不规范，使用拼音、无意义变量名
10. 存在 XSS 风险，直接渲染用户输入的原始内容
