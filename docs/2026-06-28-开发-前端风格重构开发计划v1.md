# 前端风格重构开发计划

## Context

当前前端页面采用 Bootstrap 5 默认蓝色主题 + FontAwesome 图标，视觉风格一般。根据 `docs/2026-06-27-前端-页面风格约束v1.md` 的设计规范，需要将全部 56 个 JSP 页面重构为**杂志风亮色主题**：暖白底 + 半调网点底纹、衬线标题 + 等宽数字、锗红强调色、描线 SVG 图标、自定义组件替代原生 Bootstrap 组件。

原型参考：`design-prototype/` 下 6 个 HTML 文件（`admin-dashboard.html`、`admin-user.html`、`dorm-manager-dashboard.html`、`student-dashboard.html`、`login.html`、`index.html`）已完整实现目标视觉风格，作为本次重构的视觉基准。

本次重构**仅涉及前端样式与布局**，不修改后端接口、不新增功能。

---

## 一、架构决策记录

### 决策 1：Bootstrap 保留策略

**背景**：项目重度依赖 Bootstrap 5 栅格（`row`/`col-md-*`）、工具类（`d-flex`/`mb-4`）和 JS 组件（Modal）。风格约束定义了完整自定义设计令牌体系。

| 维度 | 方案 A：保留栅格 + 覆盖组件 | 方案 B：完全移除 Bootstrap |
|:---|:---|:---|
| **未来目标支持性** | 栅格系统成熟，响应式无需重写；组件通过 CSS 覆盖可增量演进 | 完全自主可控，但需重建响应式体系 |
| **项目规范合规性** | 与现有 JSP + Bootstrap 技术栈一致，改动范围可控 | 需重写全部布局类，与课程技术栈偏离 |
| **职责边界** | 栅格/工具类=布局层，组件样式=展示层，层次清晰 | 单一职责但工作量翻倍 |

**选择**：方案 A。保留 Bootstrap 栅格和工具类，通过 CSS 变量覆盖 + 自定义类扩展替换组件样式。

### 决策 2：设计令牌体系

**背景**：Bootstrap 有 `--bs-primary` 等内置变量，风格约束定义了 `--accent`/`--fg`/`--muted` 等自定义变量。存在语义重复（如 `--bs-body-bg` vs `--bg`）。

**选择**：以风格约束的自定义变量为主变量体系（`--bg`/`--fg`/`--accent`/`--muted`/`--border` 等），同时在 `:root` 中覆盖 Bootstrap 的 `--bs-*` 变量映射到对应值。所有自定义样式引用自定义变量，Bootstrap 原生组件自动通过 `--bs-*` 覆盖获得新配色。

**理由**：自定义变量语义更清晰（`--accent` 比 `--bs-primary` 更贴近设计语言），且覆盖 `--bs-*` 可让 Bootstrap 原生组件（如 `.table-striped`、`.alert`）自动适配，减少逐组件覆盖工作量。

### 决策 3：模态框实现方式

**背景**：当前使用 Bootstrap Modal（`.modal.fade`），原型使用自定义 `.modal-overlay` + `.modal` 结构。

**选择**：逐步将 Bootstrap Modal 替换为原型定义的自定义模态框结构。理由：自定义结构与风格约束完全匹配（`#FFFFFF` 不透明背景、`translateY` 动画、`rgba(0,0,0,0.25)` 遮罩），且避免 Bootstrap Modal 的 JS 干扰。

**过渡策略**：阶段一只做样式覆盖（让 Bootstrap Modal 看起来像新风格），阶段二~四逐页替换为自定义 DOM 结构。

## 二、术语表

| 术语 | 定义 |
|:---|:---|
| 设计令牌（Design Token） | 命名的视觉属性变量（颜色、间距、圆角、字体），定义在 CSS `:root` 中，全文引用 |
| 半调网点底纹 | `body` 背景上的微小圆形网点图案（`radial-gradient`），产生纸张质感 |
| 衬线字体 | Georgia / Noto Serif SC，笔画有装饰性末端，用于 h1/h2 标题 |
| 无衬线字体 | PingFang SC / Microsoft YaHei / system-ui，笔画无装饰，用于正文 |
| 等宽字体 | SF Mono / JetBrains Mono / Consolas，每个字符等宽，用于数字/编号/标签 |
| Pill 标签 | 圆角 999px 的状态徽章，等宽字体 11px，用于表格中展示状态 |
| 幽灵按钮（Ghost Button） | 无边框无背景的轻量按钮，悬浮时显示底色，用于表格行内操作 |
| 自定义下拉选择框（cselect） | 替代原生 `<select>` 的自定义组件，由触发器 + 下拉面板 + 选项组成 |
| 数据面板（data-panel） | 表格的外层容器，白色背景 + 圆角 + 边框，承载表格和分页 |

## 三、约束条件

- 前端技术栈不变：JSP + Bootstrap 5 + jQuery 3.6 + JSTL
- Bootstrap 栅格系统和工具类保留，仅覆盖组件样式
- 禁止使用 `!important`，通过选择器层级解决优先级
- 所有颜色使用 `rgb()` / `rgba()` 格式，禁止 hex
- 设计令牌统一定义在 `:root` CSS 变量中
- 不修改后端接口
- **JS 修改边界**：仅允许修改 UI 层 JS（`renderPagination` 分页渲染、新增 `$.initCustomSelect` 自定义下拉框初始化），禁止修改业务逻辑 JS（`$.ajaxRequest`/`$.toast`/`$.confirm`/`$.formatDate` 等公共函数的现有逻辑）

## 五、组件 DOM 结构审查

### 5.1 表格（Table）

**当前**：Bootstrap `.table` + `.table-container`，`border-collapse: collapse`
**目标**：`border-collapse: separate`，列间断开竖线（`::after` 伪元素），奇数行条纹，行悬浮高亮
**原型参考**：`admin-user.html` 的 `.data-panel` 结构

**DOM 变更**：外层容器类名从 `.table-container` 改为 `.data-panel`，`<table>` 移除 Bootstrap `.table` 类

### 5.2 模态框（Modal）

**当前**：Bootstrap `.modal.fade` + `.modal-dialog`
**目标**：自定义 `.modal-overlay` + `.modal`，`#FFFFFF` 不透明背景，`translateY` 动画
**原型参考**：`admin-user.html` 的模态框结构

**DOM 变更**：逐步替换为自定义结构（阶段一样式覆盖，阶段二~四 DOM 替换）

### 5.3 按钮（Button）

**当前**：Bootstrap `.btn-primary/secondary/success/danger/warning/info`
**目标**：三类按钮 — `.btn-primary`（深色）、`.btn-secondary`（边框）、`.btn-ghost`（幽灵）
**原型参考**：`admin-user.html` 的按钮定义

**DOM 变更**：`.btn-success/danger/warning/info` 替换为 `.btn-primary/.btn-secondary/.btn-ghost`

### 5.4 状态标签（Badge）

**当前**：Bootstrap `.badge.bg-primary/success/info/warning/danger`
**目标**：自定义 `.pill-*` 系列（pill-admin/manager/student/active/pending/processing/danger/quiet/done）
**原型参考**：`admin-user.html` 的 `.pill` 定义

**DOM 变更**：全局替换 `.badge.bg-*` → `.pill-*`

### 5.5 分页（Pagination）

**当前**：Bootstrap `.pagination` 或 JS 动态渲染的分页
**目标**：左侧信息文字 + 右侧 `.page-btn` 按钮组
**原型参考**：`admin-user.html` 的 `.pagination` 结构

**DOM 变更**：`renderPagination` 函数重写，生成新 DOM 结构

### 5.6 自定义下拉选择框（cselect）

**当前**：原生 `<select>`
**目标**：自定义 `.cselect` 组件
**原型参考**：`admin-user.html` 的 `.cselect` 结构和 JS 交互

**影响范围评估**：
- 筛选区下拉框：每个列表页 1~3 个，约 30 个页面 × 平均 2 个 = ~60 处
- 表单区下拉框：每个表单页 1~3 个，约 15 个页面 × 平均 2 个 = ~30 处
- **总计约 ~90 处**原生 `<select>` 需替换

**实施策略**：阶段一在 `common.js` 中新增 `$.initCustomSelect()` 函数并提供 CSS 样式，阶段二~四逐页替换 DOM 并初始化。键盘可访问性：Tab 聚焦触发器、Enter/Space 展开面板、Escape 关闭、上下键选择。

### 5.7 其他组件覆盖

| 组件 | 变更 | 原型参考 |
|:---|:---|:---|

## 六、CSS 优先级策略

**原则**：禁止 `!important`，通过选择器层级解决优先级。

### 6.1 全局覆盖 — `:root` 变量映射

Bootstrap 原生组件通过 `--bs-*` 变量覆盖自动适配，无需额外 CSS。

### 6.2 组件覆盖 — `.app-main` 父级作用域

所有自定义组件样式通过 `.app-main`（内容区容器）提升优先级：

```css
.app-main .btn-primary { background: var(--accent); }
.app-main .data-panel td { padding: 12px var(--gap-md); }
```

### 6.3 表格容器 — `.data-panel` 类

表格样式定义在 `.data-panel` 选择器下，与 Bootstrap `.table` 解耦：

```css
.data-panel table { border-collapse: separate; border-spacing: 0; }
.data-panel th { font-family: var(--font-mono); font-size: var(--fs-sm); }
```

### 6.4 状态 — 伪类优先

竖线、hover、active 通过伪元素和伪类实现：

```css
.data-panel td:not(:last-child)::after { content: ''; ... }
.data-panel tbody tr:hover { background: var(--hover-bg); }
```

**目标**：将 FontAwesome `<i class="fas fa-xxx">` 逐步替换为内联 SVG 描线图标。
