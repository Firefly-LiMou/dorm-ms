# 高校公寓管理系统 阶段A：基础框架搭建计划V1

## 一、阶段概述

### 1.1 阶段目标
- 完成前端基础框架搭建，为后续页面开发提供统一基础
- 建立公共组件库，确保页面风格一致性
- 封装公共工具函数，提高开发效率

### 1.2 量化指标
- 公共组件复用率达到90%以上
- 页面加载时间控制在2秒以内
- 代码结构清晰，符合MVC分层规范

### 1.3 约束条件
- 严格遵循前端开发计划V1中的编码规范
- 所有第三方库必须下载到本地，禁止CDN引用
- JSP页面禁止编写Java业务逻辑
- CSS禁止使用内联样式

### 任务A-3：公共JS工具函数

**任务目标**：封装公共JS工具函数，提供统一的AJAX请求、错误处理、提示消息等功能

**文件清单**：

| 文件路径 | 说明 |
|----------|------|
| `/static/js/common.js` | 公共JS工具函数 |

**common.js函数清单**：

| 函数名 | 参数 | 返回值 | 说明 |
|--------|------|--------|------|
| `$.ajaxRequest` | url, method, data, successCallback, errorCallback | void | 统一AJAX请求封装 |
| `$.handleError` | xhr, status, error | void | 统一错误处理 |
| `$.toast` | type, message | void | 提示消息 |
| `$.confirm` | message, onConfirm, onCancel | void | 确认对话框 |
| `$.getContextPath` | 无 | string | 获取项目路径 |
| `$.navigate` | url | void | 跳转页面 |

**依赖关系**：
- 依赖jQuery核心库
- 依赖Bootstrap（用于模态框、提示框）

**验收标准**：
- 所有函数封装完成，提供JSDoc注释
- AJAX请求统一处理401/403/404/409/500错误
- 提示消息支持成功、警告、错误、信息四种类型
- 确认对话框支持自定义回调

### 任务A-4：公共布局组件

**任务目标**：创建公共布局组件（header、footer、sidebar），提供统一的页面结构

**文件清单**：

| 文件路径 | 说明 |
|----------|------|
| `/WEB-INF/jsp/common/header.jsp` | 公共头部（导航栏） |
| `/WEB-INF/jsp/common/footer.jsp` | 公共底部 |
| `/WEB-INF/jsp/common/sidebar.jsp` | 公共侧边栏（菜单） |

**header.jsp功能规划**：
- 系统Logo与标题
- 当前用户信息显示（姓名、角色）
- 修改密码按钮
- 退出登录按钮
- 响应式折叠（移动端）

**footer.jsp功能规划**：
- 版权信息
- 系统版本号

**sidebar.jsp功能规划**：
- 根据角色动态生成菜单
- 菜单项包含图标、文本、链接
- 当前页面高亮显示
- 支持多级菜单（一级菜单、二级菜单）
- 响应式折叠（移动端）

**数据来源**：
- 用户信息：从Session中获取（`${sessionScope.currentUser}`）
- 菜单数据：根据角色类型动态生成（JSTL条件判断）

**验收标准**：
- 组件可被其他JSP页面通过`<%@ include file="..." %>`或`<jsp:include page="..." />`引入
- 响应式布局在不同屏幕尺寸下正常显示
- 菜单根据角色动态生成，链接正确
- 当前页面高亮显示

## 七、风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 第三方库版本不兼容 | 页面功能异常 | 严格按照文档指定版本下载 |
| 响应式布局兼容性问题 | 移动端显示异常 | 使用Bootstrap栅格系统，充分测试 |
| 公共组件引入路径错误 | 页面无法正常显示 | 统一使用`${pageContext.request.contextPath}` |
| Session中用户信息为空 | 页面显示异常 | 添加空值判断，未登录跳转登录页 |
