# 高校公寓管理系统 前端页面逻辑设计文档V1

## 一、设计目标

### 1.1 量化指标
- 页面首屏加载时间控制在2秒以内
- AJAX请求响应处理时间控制在500毫秒以内
- 所有列表页面必须覆盖四种状态：正常数据、加载中、空数据、请求错误
- 所有表单提交必须进行前端校验，校验失败时定位到对应字段并显示红色错误提示

### 1.2 约束条件
- 前端技术栈：JSP + Bootstrap 5 + jQuery 3.6
- 认证方式：Session（Cookie）
- 视图解析器配置：前缀`/WEB-INF/jsp/`，后缀`.jsp`
- 静态资源路径：`/static/`
- 统一AJAX请求封装：`$.ajaxRequest(url, method, data, successCallback, errorCallback)`
- 统一返回格式：`{code, msg, data, extra}`

## 二、术语表

| 术语 | 说明 |
|------|------|
| 模糊查询 | 使用LIKE '%keyword%'匹配，输入任意关键字可匹配包含该关键字的记录 |
| 精确查询 | 使用=完全匹配，输入内容必须与记录完全一致 |
| 级联选择 | 选择上级选项后，下级选项动态加载对应数据 |
| 二次确认 | 执行删除、状态变更等敏感操作前，弹出确认对话框 |
| 状态徽章 | 使用Bootstrap badge组件展示状态文本，不同状态使用不同颜色 |
| 超时高亮 | 超过24小时未处理的报修记录，整行使用黄色背景标识 |
| BOM头 | UTF-8编码文件开头的特殊标记（`\uFEFF`），解析时必须移除 |

## 四、通用交互逻辑

### 4.1 列表页面通用逻辑

#### 4.1.1 页面加载流程
1. 页面加载完成，触发`$(function(){ loadData(pageQueryParams); })`
2. `loadData`函数从查询表单收集参数
3. 调用`$.ajaxRequest`发送GET请求
4. 成功回调：调用`renderTable`渲染表格，调用`renderPagination`渲染分页
5. 失败回调：显示"加载失败，点击重试"链接

#### 4.1.2 查询逻辑
- 点击"查询"按钮：重置pageNum为1，调用`loadData`
- 点击"重置"按钮：清空所有查询条件，重置pageNum为1，调用`loadData`
- 分页切换：更新pageNum，调用`loadData`
- 切换每页条数：重置pageNum为1，更新pageSize，调用`loadData`

#### 4.1.3 参数传递规范
- 空值不传递：查询条件为空时，不添加到queryParams对象中
- 数值类型转换：pageSize使用`parseInt()`转换
- 模糊查询参数名：统一使用`studentNo`（非`studentId`）

```javascript
// 正确示例
var queryParams = {
    pageNum: params.pageNum || 1,
    pageSize: params.pageSize || 10
};
var studentNo = $('#searchStudentNo').val().trim();
if (studentNo) {
    queryParams.studentNo = studentNo;
}

// 错误示例（会导致null被序列化为字符串"null"）
var queryParams = {
    studentNo: $('#searchStudentNo').val().trim() || null  // 禁止
};
```

#### 4.1.4 表格渲染规范
- 空数据：显示`<i class="fas fa-inbox mr-2"></i>暂无数据`
- 加载失败：显示`<i class="fas fa-exclamation-circle mr-2"></i>加载失败，点击重试`
- 状态字段：使用`<span class="badge bg-xxx">`展示
- 长文本截断：使用`title`属性显示完整内容

#### 4.1.5 分页渲染规范
- 分页信息：`共 X 条记录，第 X/X 页`
- 每页条数选择：10/20/50
- 页码显示：当前页前后各2页
- 首页/末页禁用：第一页禁用上一页，最后一页禁用下一页

### 4.2 表单页面通用逻辑

#### 4.2.1 表单校验规范
- 必填字段：标注红色星号`<span class="text-danger">*</span>`
- 校验失败：字段添加`is-invalid`类，下方显示红色错误信息
- 校验通过：移除`is-invalid`类，清空错误信息
- 提交按钮：提交过程中禁用并显示"提交中..."

#### 4.2.2 表单提交流程
1. 收集表单数据
2. 逐项校验，校验失败定位到对应字段
3. 校验通过，禁用提交按钮
4. 调用`$.ajaxRequest`发送POST请求
5. 成功回调：显示成功提示，1.5秒后跳转到列表页
6. 失败回调：显示错误提示，恢复提交按钮

### 4.3 宿管端页面特殊逻辑

#### 4.3.1 未分配楼栋处理
宿管端所有查询接口可能返回"您暂未负责任何楼栋"错误。页面必须处理此场景：

```javascript
$.ajaxRequest('/dorm/xxx/page', 'GET', queryParams, function(result) {
    // 正常处理
}, function(result) {
    if (result.msg && result.msg.indexOf('未负责任何楼栋') !== -1) {
        $('#tableBody').html('<tr><td colspan="X" class="text-center py-4 text-warning"><i class="fas fa-exclamation-triangle mr-2"></i>您暂未负责任何楼栋，请联系管理员</td></tr>');
    } else {
        // 显示加载失败提示
    }
});
```

### 4.4 学生选择交互逻辑

适用于：办理入住、录入晚归、录入访客

#### 4.4.1 查询流程
1. 输入学号，点击"查询"按钮
2. 调用`GET /admin/user/page?username=xxx&roleType=3&pageNum=1&pageSize=10`
3. 单条结果：直接选中，显示已选学生信息
4. 多条结果：显示查询结果列表，用户点击"选择"按钮
5. 无结果：显示"未找到学号包含 xxx 的学生"

#### 4.4.2 选择流程
1. 用户点击"选择"按钮
2. 隐藏查询结果列表
3. 显示已选学生信息（学号 + 姓名）
4. 清空查询输入框
5. 显示后续表单区域

#### 4.4.3 重新选择
1. 用户点击"重新选择"按钮
2. 清除已选学生信息
3. 隐藏已选学生信息区域
4. 隐藏后续表单区域

#### 4.4.4 HTML结构规范
查询结果和已选信息必须使用独立的div，禁止混用：

```html
<!-- 查询结果列表 -->
<div id="studentResult" style="display: none;">
    <table>...</table>
</div>
<!-- 已选学生信息 -->
<div id="selectedStudent" style="display: none;">
    <div class="alert alert-success">
        已选学生：<strong id="selectedStudentInfo"></strong>
        <button onclick="clearSelectedStudent()">重新选择</button>
    </div>
</div>
```

### 4.6 文件上传逻辑

适用于：批量导入用户

#### 4.6.1 前端校验
1. 文件非空校验
2. 文件扩展名校验（.csv）
3. 文件大小校验（1MB）

#### 4.6.2 上传流程
1. 创建FormData对象
2. 调用`$.ajax`发送POST请求（不能使用`$.ajaxRequest`）
3. 设置`processData: false, contentType: false`
4. 成功回调：显示导入结果
5. 失败回调：显示错误提示

```javascript
var formData = new FormData();
formData.append('file', file);

$.ajax({
    url: $.buildUrl('/admin/user/import'),
    type: 'POST',
    data: formData,
    processData: false,
    contentType: false,
    dataType: 'json',
    success: function(result) { ... },
    error: function(xhr, status, error) { ... }
});
```

## 五、模块交互流程

### 5.1 用户认证模块

#### 5.1.1 登录流程
```
用户访问系统 → 跳转login.jsp
→ 输入用户名、密码 → 前端校验非空
→ POST /login（使用$.ajax发送JSON）
→ 成功（code=200）：
  → 检查extra.needChangePassword
  → true：弹出修改密码模态框（不可关闭）
  → false：根据roleType跳转对应首页
→ 失败（code=401）：显示"用户名或密码错误"或"账号已被禁用"
```

#### 5.1.2 首次登录强制修改密码
```
登录成功后检测到needChangePassword=true
→ header.js自动弹出模态框（backdrop:static, keyboard:false）
→ 输入新密码（6-20位）+ 确认密码
→ 前端校验：非空、一致性、长度
→ POST /updatePassword → {newPassword, confirmPassword}
→ 成功：提示"修改成功"，跳转登录页
→ 失败：显示错误信息，模态框保持打开
```

#### 5.1.3 退出登录流程
```
点击导航栏"退出登录"
→ $.confirm二次确认
→ POST /logout
→ 清除sessionStorage(sidebarMenuExpanded)
→ 提示"退出成功"
→ 1秒后跳转/login
```

### 5.2 用户管理模块

#### 5.2.1 用户列表与查询流程
```
管理员访问 /admin/user/list
→ 页面加载，自动查询第一页 GET /admin/user/page
→ 可选条件：用户名（模糊）、姓名（模糊）、角色类型、状态

→ 点击"重置"：清空条件，重新查询全部
→ 表格列：用户名、姓名、角色、性别、年级、专业、班级、手机号、状态、操作

```

#### 5.2.2 新增用户流程
```
点击"新增用户" → 跳转addPage
→ 填写表单：用户名（必填）、姓名（必填）、角色类型（下拉1/2/3）、性别（下拉）、手机号（11位）、年级/专业/班级（角色=学生时必填）

→ POST /admin/user/add
→ 成功：提示"新增成功"，跳转列表页
→ 失败（用户名冲突409）：提示"用户名已存在"
```

```
列表页点击"编辑" → 跳转editPage?userId=xxx
→ 页面加载：GET /admin/user/detail/{userId}
→ 回填表单各字段
→ 修改后提交：POST /admin/user/update
→ 成功：提示"修改成功"，跳转列表页
```

#### 5.2.4 启用/禁用用户流程
```
列表页点击"启用/禁用"按钮
→ $.confirm二次确认
→ POST /admin/user/toggleStatus/{userId}
→ 成功：提示"操作成功"，刷新列表
```

## 六、数据流向

### 6.1 页面加载流程
```
用户访问URL → DispatcherServlet
→ LoginInterceptor（校验Session）
→ PermissionInterceptor（校验角色权限）
→ Controller（返回视图名称）
→ ViewResolver（拼接前缀+视图名+后缀）
→ JSP页面渲染（结合Model数据）
→ 响应返回浏览器
```

### 6.2 AJAX请求流程
```
用户触发交互（点击按钮、提交表单）
→ JavaScript收集数据
→ 前端校验
→ 调用$.ajaxRequest发送请求
→ 后端处理请求，返回JSON结果
→ 前端处理响应：
  - 成功（code=200）：更新页面、显示成功提示
  - 未登录（code=401）：跳转登录页
  - 无权限（code=403）：显示无权限提示
  - 其他错误：显示错误信息
```

### 6.3 分页查询流程
```
页面加载 → 初始化分页参数（pageNum=1, pageSize=10）
→ 调用查询函数
→ 后端执行分页查询，返回PageInfo对象
→ 前端渲染数据表格
→ 前端渲染分页组件
→ 用户点击页码 → 更新pageNum → 重新调用查询函数
```
