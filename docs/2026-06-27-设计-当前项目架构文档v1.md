# 高校公寓管理系统 当前项目架构文档V1

## 一、设计目标

### 1.1 量化指标
- 单接口响应时间控制在1秒以内
- 支持50人同时在线访问
- 核心业务方法测试覆盖率达到80%以上

### 1.2 约束条件
- JDK版本：11
- 技术栈：Spring 5.3.31 + Spring MVC 5.3.31 + MyBatis 3.5.16 + Maven + Tomcat 9.x + MySQL 8.0
- 架构模式：单体三层架构（Controller → Service → DAO）
- 部署方式：WAR包部署至Tomcat
- 前端方案：JSP + Bootstrap + jQuery

## 三、整体分层架构

### 3.1 架构总览

```
┌───────────────────────────────────────────────────┐
│                   视图层 (View)                    │
│  JSP + Bootstrap + jQuery + JSTL                  │
└─────────────────┬─────────────────────────────────┘
                  │ HTTP请求/响应
┌─────────────────▼─────────────────────────────────┐
│                 表现层 (Controller)                │
│  Spring MVC控制层：请求接收、参数校验、结果封装    │
│  按角色分包：admin/ dorm/ student/                │
└─────────────────┬─────────────────────────────────┘
                  │ 方法调用
┌─────────────────▼─────────────────────────────────┐
│                  业务层 (Service)                  │
│  接口 + 实现类：业务逻辑、事务控制、状态流转      │
└─────────────────┬─────────────────────────────────┘
                  │ 数据访问
┌─────────────────▼─────────────────────────────────┐
│                 持久层 (DAO/Mapper)                │
│  MyBatis Mapper接口 + XML：SQL映射、CRUD操作       │
└─────────────────┬─────────────────────────────────┘
                  │ JDBC连接
┌─────────────────▼─────────────────────────────────┐
│                  数据层 (Database)                 │
│  MySQL 8.0：10张业务表，InnoDB引擎，utf8mb4       │
└───────────────────────────────────────────────────┘

【横切公共组件（Spring AOP / 拦截器）】
┌───────────────────────────────────────────────────┐
│  登录拦截器、权限拦截器、操作日志切面、全局异常处理│
└───────────────────────────────────────────────────┘
```

### 3.2 各层职责划分

| 层级 | 核心职责 | 技术实现 | 约束 |
|------|----------|----------|------|
| Controller | 接收请求、参数校验、调用Service、封装返回 | @Controller、@ResponseBody、@Valid | 禁止编写业务逻辑、禁止直接操作数据库 |
| Service | 业务逻辑、事务控制、业务参数校验 | @Service、@Transactional | 禁止处理HTTP请求、禁止拼接SQL |
| DAO | 数据库CRUD、SQL映射 | Mapper接口 + XML | 禁止编写业务逻辑、SQL必须预编译 |
| 横切组件 | 登录校验、权限拦截、操作日志、异常处理 | AOP、拦截器、@ControllerAdvice | 与业务代码解耦 |

```java
public class Result<T> {
    private Integer code;           // 状态码
```

**静态工厂方法**：
- `success()` / `success(data)` / `success(msg, data)`
- `fail(msg)` / `fail(code, msg)`
- `badRequest(msg)` / `unauthorized(msg)` / `forbidden(msg)`
- 链式调用 `putExtra(key, value)`

## 六、操作审计设计（课程亮点）

### 6.1 实现方式

自定义`@OperLog`注解 + AOP环绕通知，自动记录管理端增删改操作。

### 6.2 注解定义

```java
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface OperLog {
    String module();           // 操作模块
    OperTypeEnum type();       // 操作类型
    String desc() default "";  // 操作描述
}
```

### 6.3 切面执行流程

```
1. 拦截@OperLog注解的方法
2. 获取方法上的注解信息（module、type、desc）
3. 获取HttpServletRequest请求对象
4. 从Session获取操作人信息（SessionUtil.getCurrentUser）
5. 获取操作IP（依次尝试X-Forwarded-For、X-Real-IP、getRemoteAddr）
6. 获取请求参数（JSON序列化，截取前1000字符）
7. 执行目标方法（@Around的proceed）
8. 构建SysOperLog对象

```

## 七、拦截器设计

### 7.1 拦截器链执行顺序

```
请求进入 → DispatcherServlet
    → LoginInterceptor（登录校验）
    → PermissionInterceptor（权限校验）
    → Controller方法
```

### 7.2 登录拦截器（LoginInterceptor）

**职责**：校验用户是否已登录

**实现逻辑**：
- 从Session获取当前用户（SessionUtil.getCurrentUser）
- 用户为null时，重定向至`/login`，返回false
- 用户存在时，放行请求，返回true

**拦截范围**：`/**`（全部路径）

**排除路径**：
- `/login`（登录页、登录接口）
- `/logout`（登出接口）
- `/static/**`（静态资源）
- `/captcha`（验证码接口）

### 7.3 权限拦截器（PermissionInterceptor）

**职责**：基于URL路径前缀校验角色权限

**实现逻辑**：
- 使用`AntPathMatcher`匹配请求路径
- 从Session获取当前用户的角色类型
- 校验路径与角色的对应关系：
  - `/admin/**` → 仅允许 roleType=1 (ADMIN)
  - `/dorm/**` → 仅允许 roleType=2 (DORM_MANAGER)
  - `/student/**` → 仅允许 roleType=3 (STUDENT)
- 角色不匹配时，返回403 Forbidden

**拦截范围**：分别匹配 `/admin/**`、`/dorm/**`、`/student/**`

## 八、用户认证设计

### 8.1 认证流程

```
用户提交账号密码 → AuthController.login()
    → 调用UserService.login(LoginDTO)
    → 校验用户名是否存在（SysUserMapper.selectByUsername）
    → 校验密码是否正确（Md5Util.verify）
    → 校验账号状态是否正常（UserStatusEnum.NORMAL）
    → 将用户信息存入Session（SessionUtil.setCurrentUser）
    → 清除密码字段后返回用户信息
    → 返回Result.success(user)
```

### 8.2 首次登录强制修改密码

**判断逻辑**：比对密码是否等于默认密码（年级+手机号的MD5值）

**登录接口返回**：
```json
{
  "code": 200,
  "data": { "userId": 1, "username": "admin", "roleType": 1 },
  "extra": { "needChangePassword": true }
}
```

### 8.3 密码修改

- 校验新密码与确认密码一致
- 校验新密码长度6-20位
- 使用MD5加密后存储

## 十三、安全设计

### 13.1 数据安全

- 密码MD5加密存储，禁止明文
- 敏感数据脱敏输出（VisitorVO中身份证号脱敏）
- 核心业务表禁止物理删除，采用逻辑删除

### 13.2 访问安全

- 所有业务接口必须经过登录拦截器校验
- 基于角色的接口级权限校验（PermissionInterceptor）
- 学生查询个人数据强制从Session获取用户ID，禁止传参指定他人

### 13.3 输入安全

- MyBatis全部使用预编译语句`#{}`，杜绝SQL注入
- 所有入参通过Hibernate Validator注解校验
- 请求参数序列化时截取前1000字符，防止恶意大数据

## 十四、请求处理全流程

```
1. 客户端发起HTTP请求
2. Tomcat接收，转发至DispatcherServlet
3. 执行CharacterEncodingFilter，设置UTF-8编码
4. 执行LoginInterceptor：校验Session中是否存在用户
5. 执行PermissionInterceptor：校验路径与角色匹配
6. 进入Controller层：参数绑定、@Valid校验
7. 调用Service层：执行业务逻辑、事务控制
8. Service调用DAO层：执行MyBatis Mapper
9. DAO层执行SQL：通过Druid连接池获取连接
10. 返回结果链：DAO → Service → Controller
11. Controller封装Result返回
12. OperLogAspect记录操作日志（如方法标注@OperLog）
13. GlobalExceptionHandler捕获异常（如发生）
14. 响应返回客户端
```
