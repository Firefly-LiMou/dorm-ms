# 高校公寓管理系统 后端开发计划V1

## 一、设计目标

### 1.1 量化指标
- 项目必须在14个工作日内完成全部必做模块开发与联调
- 单接口响应时间必须控制在1秒以内
- 系统必须支持50人同时在线访问
- 代码测试覆盖率必须达到核心业务方法80%以上

### 1.2 约束条件
- JDK版本：11
- 技术栈：Spring 5.3.x + Spring MVC + MyBatis 3.5.x + Maven + Tomcat 9.x + MySQL 8.0
- 架构模式：单体三层架构（Controller → Service → DAO）
- 部署方式：WAR包部署至Tomcat

### 阶段二：公共组件开发（第2-3天）

#### 3.2.1 统一返回结果封装
**包路径**：`com.h.dormitory.common.result`

**类设计**：
```
Result<T>
├── code: Integer        // 状态码
├── msg: String          // 提示信息
├── data: T              // 返回数据
├── success(): Result    // 静态方法，返回成功结果
├── fail(String): Result // 静态方法，返回失败结果
└── fail(Integer, String): Result // 静态方法，返回自定义错误码
```

**状态码规范**：
- 200：操作成功
- 400：参数错误
- 401：未登录
- 403：无权限
- 500：系统内部错误
- 1000+：业务自定义错误码

#### 3.2.2 自定义异常类
**包路径**：`com.h.dormitory.common.exception`

**类设计**：
```
BusinessException extends RuntimeException
├── code: Integer        // 业务错误码
├── msg: String          // 错误提示信息
├── constructor(Integer, String)
└── constructor(String)
```

#### 3.2.3 全局异常处理器
**包路径**：`com.h.dormitory.common.exception`

**类设计**：
```
GlobalExceptionHandler (@ControllerAdvice)
├── handleBusinessException(BusinessException): Result
├── handleMethodArgumentNotValid(MethodArgumentNotValidException): Result
└── handleException(Exception): Result
```

**处理规则**：
- BusinessException：返回业务错误码与提示信息

```
└── getCheckinListByBuildingId(Long buildingId, Integer pageNum, Integer pageSize): PageInfo<CheckinVO>

MoveApplyService
├── apply(MoveApplyDTO, Long studentId): void
├── audit(Long applyId, MoveAuditDTO, Long auditorId): void
├── getApplyById(Long applyId): MoveApplyVO
├── getMyApplies(Long studentId): List<MoveApplyVO>
├── getApplyList(DormMoveApply query, Integer pageNum, Integer pageSize): PageInfo<MoveApplyVO>
└── getPendingList(Integer pageNum, Integer pageSize): PageInfo<MoveApplyVO>
```

**业务规则（必须使用@Transactional）**：
- checkin：
  1. 校验学生当前无在住记录
  2. 校验目标床位状态为空闲
  3. 事务内执行：插入入住记录 + 更新床位状态为已入住
- checkout：
  1. 校验入住记录存在且状态为在住
  2. 事务内执行：更新入住记录状态为已退宿 + 更新床位状态为空闲
- audit（通过时）：
  1. 校验目标床位状态为空闲
  2. 事务内执行：原床位置空闲 + 目标床位置已入住 + 更新原入住记录 + 创建新入住记录 + 更新申请状态

```
│   - 获取请求参数（JSON序列化）
│   - 执行目标方法
│   - 异步写入sys_oper_log表
│   - 返回目标方法执行结果
```

## 四、分层职责与边界

### 4.1 Controller层职责
- 接收前端请求，完成参数绑定与基础格式校验
- 调用Service层处理业务
- 封装统一Result返回结果
- 添加@OperLog注解记录操作日志

### 4.2 Controller层禁止项
- 禁止直接操作数据库
- 禁止编写复杂业务逻辑
- 禁止直接输出异常堆栈

### 4.3 Service层职责
- 实现核心业务逻辑，处理状态流转
- 调用DAO层进行数据操作
- 使用@Transactional声明式事务管理
- 业务参数合法性校验

### 4.4 Service层禁止项
- 禁止直接处理HTTP请求与响应
- 禁止拼接SQL
- 禁止跨层调用Controller

### 4.5 DAO层职责
- 封装数据库CRUD操作
- 编写MyBatis XML映射文件
- 执行动态SQL与多表关联查询

### 4.6 DAO层禁止项
- 禁止编写业务逻辑
- 禁止处理业务异常
- 禁止使用${}拼接非必要字段

## 五、数据流向

### 5.1 请求处理流程
```
HTTP请求 → DispatcherServlet → 拦截器链（登录校验 → 权限校验）
    → Controller（参数绑定、基础校验）
    → Service（业务逻辑、事务控制）
    → DAO（SQL执行）
    → 数据库
    → 返回结果链
    → Controller封装Result
    → 响应前端
```

### 5.2 操作日志记录流程
```
Controller方法标注@OperLog → AOP切面拦截
    → 获取操作人、模块、类型、参数、IP
    → 执行目标方法
    → 异步写入sys_oper_log
    → 返回目标方法结果
```
