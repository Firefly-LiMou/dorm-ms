About the Project

为高校公寓日常管理场景设计

覆盖入住登记、调宿审批、报修处理、晚归登记、访客管理等完整业务流程。系统采用 MVC 三层架构，WAR 包部署，面向系统管理员、宿管、学生三种角色提供差异化界面与权限控制。

01 — 项目背景

解决传统纸质管理的低效与混乱

传统公寓管理依赖纸质登记和人工传递信息，数据分散、查询困难、审批流程漫长。系统将全部流程线上化，实现数据集中管理与实时查询。

02 — 技术架构

Spring MVC + MyBatis + MySQL

后端采用 Spring MVC 5.3 + MyBatis 3.5 + MySQL 8.0，前端 Bootstrap 5 + jQuery。MVC 三层架构，WAR 包部署。

Spring MVC 5.3 MyBatis 3.5 MySQL 8.0 Bootstrap 5 jQuery

03 — 工程规范

企业级开发标准

统一返回格式、全局异常处理、AOP 操作日志、声明式事务、枚举替代魔法值、SQL 预编译防注入。

Highlights

六大核心亮点

01

三角色权限隔离

管理员、宿管、学生三套独立界面，接口级拦截器 + 数据级楼栋隔离，确保各角色只能访问授权范围内的数据。

02

Spring AOP 操作日志

无侵入自动记录所有管理端增删改操作，支持审计追溯，每次操作精确到操作人、时间、内容与 IP。

03

事务一致性保障

入住、调宿、退宿等核心操作使用 @Transactional 原子执行，数据变更要么全部成功，要么全部回滚。

04

批量导入用户

CSV 文件一键导入，逐行校验 + 部分成功策略，导入结果精确反馈每行状态，支持大规模新生入住数据录入。

05

数据看板

ECharts 可视化统计，入住率趋势、报修分布、楼栋对比一目了然，辅助管理者快速决策。

06

标准化文档体系

需求、架构、接口、业务逻辑、页面逻辑、测试验收全链路文档，确保项目可维护、可交接。
